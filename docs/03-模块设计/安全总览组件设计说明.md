## 1. 业务概述与对外接口 (Overview & Public Interfaces)

- **核心目标**: 安全总览作为应用首页，聚合企业安全管理关键状态，并提供进入各业务模块的快捷入口。
- **入口路由 (Route Entry)**: `RouteIds.DASHBOARD`，由 `entry/src/main/ets/pages/MainPage.ets` 挂载到 `DashboardPage`。
- **对外暴露能力 (Public APIs)**: 无。安全总览不向其它模块暴露 Service 或写操作接口，只作为页面展示与摘要聚合模块。
- **业务边界**:
  - ✅ **包含**:
    - 展示企业管理员激活状态摘要。
    - 展示防火墙启停状态和当前模式摘要。
    - 展示日志今日数量和累计数量摘要。
    - 展示外设连接记录总数、连接次数和断开次数摘要。
    - 展示防火墙管理、日志管理、外设管理、身份鉴别、工具设置、帮助与反馈六个快捷入口。
    - 处理页面初始化、页面重新显示刷新、日志变化局部刷新、外设连接记录变化局部刷新。
    - 将快捷入口点击转换为标准页面路由跳转。
  - ❌ **不包含**:
    - 不激活或停用企业管理员能力。
    - 不编辑防火墙总开关、模式、规则或域名/IP 策略。
    - 不管理日志明细、分页、清理、导出或存储配置。
    - 不管理外设策略、接口控制或连接记录明细。
    - 不承载身份鉴别、工具设置、帮助与反馈的业务逻辑。
    - 不作为管理员、防火墙、日志、外设等模块的最终业务真相源。

## 2. 状态与数据流 (Data Flow & State)

> **设计原则**: 单向数据流 (View -> ViewModel -> Service -> Repository/Storage)

- **核心业务状态 (Core Business State)**:
  - `moduleCards: DashboardModuleCardState[]`: 安全总览四张摘要卡状态，初始为空数组，初始化刷新成功后固定包含 `admin-status`、`firewall`、`log-manage`、`peripheral-manage`。
  - `quickEntries: DashboardQuickEntryState[]`: 安全总览快捷入口状态，初始为空数组，初始化时优先构建，当前固定包含 `firewall`、`log-manage`、`peripheral-manage`、`identity`、`tool-settings`、`help-feedback`。
  - `context: common.UIAbilityContext | null`: 页面宿主上下文，初始为 `null`。用于读取防火墙概览状态和初始化外设连接记录维护服务。
  - `logListenerId: number`: 日志仓库变更监听句柄，初始为 `-1`。大于 0 表示已注册。
  - `peripheralTraceListenerId: number`: 外设连接记录仓库变更监听句柄，初始为 `-1`。大于 0 表示已注册。

- **关键流转路径**:
  - `DashboardPage.aboutToAppear()` -> `initializePage()` -> `DashboardViewModel.initialize(context)` -> 优先构建快捷入口、全量刷新摘要卡、注册监听 -> `moduleCards` / `quickEntries` 驱动 UI 渲染；若页面初始化链路异常，页面记录 warn 日志并继续播放入场动画。
  - `DashboardPage.onPageShow()` -> `DashboardViewModel.refreshStaticStatuses()` -> 当前实现调用 `refreshAll()` -> 重新构建四张摘要卡 -> 页面状态校准。
  - `LogEntryRepository` 变更通知 -> `refreshLiveCards(false, true)` -> 仅重建并替换日志卡 -> UI 局部刷新日志摘要。
  - `PeripheralTraceEntryRepository` 变更通知 -> `refreshLiveCards(true, false)` -> 仅重建并替换外设卡 -> UI 局部刷新外设摘要。
  - `ToolCard.onPress` -> `DashboardPage.onNavigate(entry.route)` -> `MainPage.setCurrentPage(route)` -> 主页面路由切换。
  - `DashboardPage.aboutToDisappear()` -> `DashboardViewModel.destroy()` -> 注销日志和外设监听 -> 清理页面动画 timer。

## 4. 模块结构与组件设计 (Module Components)

### 【核心层】(Core MVVM & Domain Layers)

#### 4.1 Model & Types (核心数据模型与类型)

* **核心实体与依赖路径**:
  * `DashboardModuleCardState`
    * **所在文件**: `entry/src/main/ets/models/DashboardModels.ets`
    * **业务作用**: 描述安全总览摘要卡的 UI 状态，是管理员、防火墙、日志、外设四类首页摘要的统一展示模型。
    * **关键字段**: `id` (当前取值为 `admin-status`、`firewall`、`log-manage`、`peripheral-manage`), `title` (摘要卡标题), `primaryText` (主状态或主指标), `secondaryText` (辅助说明), `accentStart` / `accentEnd` (图标渐变色), `icon` (图标资源)。
  * `DashboardQuickEntryState`
    * **所在文件**: `entry/src/main/ets/models/DashboardModels.ets`
    * **业务作用**: 描述安全总览快捷入口的展示状态和路由目标。
    * **关键字段**: `id` (当前取值为 `firewall`、`log-manage`、`peripheral-manage`、`identity`、`tool-settings`、`help-feedback`), `title` (入口标题), `route` (点击后进入的页面路由), `icon` / `accentStart` / `accentEnd` (展示资源)。

#### 4.2 Service / Domain (领域业务层)

* **真实文件路径**: 无独立 Service。安全总览是摘要聚合模块，核心业务用例由 `DashboardViewModel` 编排。
* **核心业务用例 (Use Cases)**:
  * `EnterpriseAdminService.probeActivation()`: 读取企业管理员激活状态，用于生成管理员状态卡。
    * **副作用**: 无写入副作用。
    * **失败策略**: `DashboardViewModel.buildAdminCard()` 捕获异常，记录 warn 日志，并降级展示 `未激活`。
  * `FirewallService.loadOverviewState(context)`: 读取防火墙启停状态和当前模式，用于生成防火墙模式卡。
    * **副作用**: 无写入副作用。
    * **失败策略**: `DashboardViewModel.buildFirewallCard()` 捕获异常，记录 warn 日志，并降级展示 `已禁用` 与默认模式标题。
  * `LogEntryRepository.getStatistics()`: 读取日志统计，用于生成日志记录卡。
    * **副作用**: 无写入副作用。
    * **失败策略**: `DashboardViewModel.buildLogCard()` 捕获异常，记录 warn 日志，并降级展示今日 0 条、累计 0 条，避免日志仓库未就绪阻断首页渲染。
  * `LogEntryRepository.addChangeListener(...)` / `removeChangeListener(...)`: 注册和注销日志统计变更监听。
    * **副作用**: 维护仓库监听器注册状态。
    * **失败策略**: `destroy()` 仅在 listener id 大于 0 时注销，允许未初始化时安全调用。
  * `PeripheralTraceMaintenanceService.initialize(context)`: 初始化外设连接记录维护能力。
    * **副作用**: 初始化外设连接记录相关存储或维护服务。
    * **失败策略**: 返回失败时只记录 warn，不阻断安全总览继续渲染。
  * `PeripheralTraceEntryRepository.getStatistics()`: 读取外设连接记录统计，用于生成外设连接记录卡。
    * **副作用**: 无写入副作用。
    * **失败策略**: `DashboardViewModel.buildPeripheralCard()` 捕获异常，记录 warn 日志，并降级展示累计 0 条、连接 0 次、断开 0 次。
  * `PeripheralTraceEntryRepository.addChangeListener(...)` / `removeChangeListener(...)`: 注册和注销外设连接记录变更监听。
    * **副作用**: 维护仓库监听器注册状态。
    * **失败策略**: `destroy()` 仅在 listener id 大于 0 时注销，允许未初始化时安全调用。
* **内部架构设计**:
  * **Repository**: 安全总览无自有 Repository。日志摘要来自 `entry/src/main/ets/services/log-manage/repository/LogEntryRepository.ets`，外设摘要来自 `entry/src/main/ets/services/peripheral/connection-record/PeripheralTraceEntryRepository.ets`。
  * **Mapper/Ports**: 无独立 Mapper。摘要卡和快捷入口由 `DashboardViewModel` 直接从外部只读摘要结果转换为 `DashboardModuleCardState` 和 `DashboardQuickEntryState`。

#### 4.3 ViewModel (视图模型层)

* **真实文件路径**: `entry/src/main/ets/viewmodels/dashboard/overview/DashboardViewModel.ets`
* **状态分发逻辑**:
  * `initialize(context)`: 保存页面上下文，先构建六个快捷入口，再初始化外设连接记录维护服务，执行一次全量摘要刷新，然后注册日志和外设变更监听。外部摘要读取失败由对应卡片降级处理，不得阻断其它摘要读取。
  * `refreshAll()`: 调用 `buildModuleCards()` 重建四张摘要卡，并整体更新 `moduleCards`；若全量刷新异常，则使用默认四卡兜底。
  * `refreshStaticStatuses()`: 页面重新显示时的状态校准入口。当前实现直接调用 `refreshAll()`，因此管理员、防火墙、日志、外设都会重新读取。
  * `registerListeners()`: 注册日志和外设连接记录变更监听。方法会检查 listener id，避免重复注册。
  * `refreshLiveCards(refreshPeripheral, refreshLog)`: 监听回调使用的局部刷新入口。根据触发来源只替换日志卡或外设卡，避免不必要地重建其它摘要卡。
  * `destroy()`: 注销日志和外设监听，并将 listener id 重置为 `-1`。
  * `buildAdminCard()`: 通过 `EnterpriseAdminService.probeActivation()` 生成管理员状态卡。激活时展示 `已激活` 和 `企业管理能力已激活`，未激活或读取失败时展示 `未激活` 和 `企业管理能力未激活`。
  * `buildFirewallCard()`: 通过 `FirewallService.loadOverviewState(context)` 生成防火墙模式卡。启用时展示 `已启用`，关闭或读取失败时展示 `已禁用`；副说明展示当前模式标题。
  * `buildLogCard()`: 通过 `LogEntryRepository.getStatistics()` 生成日志记录卡。主值为 `todayCount`，副说明为累计 `totalCount`；读取失败时降级为 0。
  * `buildPeripheralCard()`: 通过 `PeripheralTraceEntryRepository.getStatistics()` 生成外设连接记录卡。主值为 `total`，副说明为连接次数和断开次数；读取失败时降级为 0。
  * `buildQuickEntries()`: 固定生成防火墙管理、日志管理、外设管理、身份鉴别、工具设置、帮助与反馈六个快捷入口。

#### 4.4 View / Page (页面视图层)

* **真实文件路径**: `entry/src/main/ets/views/dashboard/overview/DashboardPage.ets`
* **结构说明**:
  * `DashboardPage`: 安全总览页面入口。页面接收 `currentTheme` 和 `onNavigate`，内部持有 `DashboardViewModel`。
  * 标题区: 展示 `ModuleText.DASHBOARD` 和 `DashboardPageStrings.SUBTITLE`。
  * 摘要卡区: 使用 `GridRow({ columns: 4, gutter: 22 })` 渲染 `viewModel.moduleCards`，每张卡使用 `MetricSummaryCard`。当前摘要卡配置为 `clickable: false`、`hoverable: true`，只承担状态展示，不承担页面跳转。
  * 快捷入口标题: 展示 `DashboardPageStrings.QUICK_ACTIONS_TITLE`。
  * 快捷入口区: 使用 `GridRow({ columns: 3, gutter: 22 })` 渲染 `viewModel.quickEntries`，每个入口使用 `ToolCard`。点击时调用 `onNavigate(entry.route)`，由 `MainPage` 执行页面切换。
  * 页面生命周期: `aboutToAppear()` 触发初始化，`onPageShow()` 触发状态刷新，`aboutToDisappear()` 销毁监听并清理动画 timer。初始化异常时页面记录 warn 日志并继续播放入场动画，不额外补齐 ViewModel 状态。
  * 页面动画: 标题、摘要卡、快捷方式标题和快捷入口分批进入。摘要卡和快捷入口动画数组长度必须由 `viewModel.moduleCards.length`、`viewModel.quickEntries.length` 派生，避免卡片数量变化时出现动画状态越界或漏播；动画状态是页面级临时 UI 状态，不进入 ViewModel。

#### 4.5 Components (可复用组件层)

* **真实文件路径**: `entry/src/main/ets/components/MetricSummaryCard.ets`、`entry/src/main/ets/components/ToolCard.ets`
* **结构说明**:
  * `MetricSummaryCard`: 安全总览摘要卡复用组件。**入参/事件**: 接收 `title`、`primaryText`、`secondaryText`、`icon`、`accentStart`、`accentEnd`、`currentTheme`、`clickable`、`hoverable`、`onPress` 等。安全总览当前只使用其展示和 hover 能力，不使用点击跳转能力。
  * `ToolCard`: 安全总览快捷入口复用组件。**入参/事件**: 接收 `label`、`icon`、`gradientStart`、`gradientEnd`、`currentTheme` 和 `onPress`。安全总览通过 `onPress` 将入口点击转交给 `DashboardPage.onNavigate`。

## 5. 异常处理与系统依赖 (Dependencies & Errors)

- **关键系统 API**:
  - `@kit.AbilityKit/common.UIAbilityContext`: 用于安全总览初始化时传递页面宿主上下文。
- **系统权限**:
  - 安全总览模块自身不新增独立权限。
  - 管理员、防火墙、日志、外设等被聚合模块的系统权限由各自模块和应用清单承担，安全总览不在本模块内展开这些权限配置。
- **异常兜底策略**:
  - 外设连接记录维护初始化失败时，`DashboardViewModel.initialize()` 记录 warn 日志，但继续构建快捷入口和摘要卡，不阻断首页渲染。
  - 管理员激活状态读取失败时，`buildAdminCard()` 捕获异常，记录 warn 日志，并降级展示未激活状态。
  - 防火墙概览状态读取失败时，`buildFirewallCard()` 捕获异常，记录 warn 日志，并降级展示 `已禁用` 和默认模式标题。
  - 日志统计或外设统计读取失败时，对应卡片捕获异常并降级为 0 统计；日志仓库清理安装后暂未初始化也不得阻断首页其它区域展示。
  - 日志或外设监听触发的局部刷新失败时，`refreshLiveCards()` 捕获异常，记录 warn 日志，并保留已有 `moduleCards`。
  - 页面初始化链路异常时，`DashboardPage.initializePage()` 记录 warn 日志并继续执行入场动画；页面不额外补齐 ViewModel 状态。
  - `destroy()` 可在未完成初始化或监听未注册时调用；只有 listener id 大于 0 时才执行注销。
  - 摘要卡当前不承担业务写操作，因此首页交互失败面主要集中在路由跳转和外部摘要读取。

### 5.1 实施步骤与测试验收 (Implementation & Acceptance)

- **实施步骤**:
  - 涉及摘要卡、快捷入口、首页路由或聚合状态展示的变更，必须先更新本文档中的目标范围、页面结构、状态模型、职责边界和异常兜底口径。
  - 静态文案、图标、颜色或入口顺序先同步 `DashboardModels.ets`、`DashboardStrings.ets`、`AppResources.ets`、`RouteIds.ets` 和 `StringCatalog.ets`，再调整页面渲染。
  - 聚合状态读取或监听逻辑先在 `DashboardViewModel.ets` 内收敛，再由 `DashboardPage.ets` 只消费 `DashboardPageState`；页面不得直接调用防火墙、日志、外设等业务服务。
  - 新增快捷入口时，需要同时检查 `MainPage.ets` 路由映射、顶部/侧边导航入口、E2E 导航用例和对应模块设计文档。
- **测试覆盖**:
  - UT: `entry/src/test/dashboard/viewmodel.test.ets` 覆盖摘要卡构造、快捷入口构造、外部状态降级和监听刷新。
  - ohosTest: `entry/src/ohosTest/ets/test/simple/RouteAction.test.ets` 通过 `route_action` 场景覆盖安全总览入口和跨模块导航基线。
  - E2E: `scripts/e2e/cases/dashboard/overview.json`、`summary_cards.json`、`quick_entries.json`、`admin_status.json` 覆盖首页加载、卡片展示、快捷入口和管理员状态。
- **验收口径**:
  - 首页默认进入 `dashboard`，四类摘要卡和六个快捷入口稳定展示，快捷入口能跳转到目标模块。
  - 防火墙、日志、外设任一外部摘要读取失败时，首页不白屏、不阻断其它卡片展示，并保留可解释的降级状态。
  - 首页不新增独立系统权限，不绕过各业务模块已有的权限、企业管理员和认证边界。
