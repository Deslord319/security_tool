## 1. 业务概述与对外接口 (Overview & Public Interfaces)
- **核心目标**: 提供企业管理员场景下的系统口令复杂度与密码有效期策略配置能力，并沉淀可被其它敏感操作复用的 PIN / 指纹本地认证能力。
- **入口路由 (Route Entry)**: `RouteIds.IDENTITY = 'identity'` -> `entry/src/main/ets/views/identity/settings/IdentityPage.ets`
- **对外暴露能力 (Public APIs)**:
  - `IdentityService`: 口令策略读取、保存。
  - `IdentityPasswordPolicyMapper`: `PasswordComplexityConfig` 与系统 `PasswordPolicy` 的双向转换、复杂度正则构建和策略描述构建。
  - `AuthService`: PIN / 指纹可用性检查与系统认证执行。
  - `UserAuthAdapter`: `@kit.UserAuthenticationKit` 的薄适配层，供 `AuthService` 调用并便于测试。
- **业务边界**:
  - ✅ **包含**: 最小密码长度、大小写字母、数字、特殊字符、密码有效期策略配置；企业管理员状态前置感知；通过 `IdentityPasswordPolicyMapper` 完成口令策略与 MDM `PasswordPolicy` 的双向转换；PIN / 指纹认证基础能力封装。
  - ❌ **不包含**: 企业管理员激活流程本身；账号体系、统一身份平台、账号锁定/失败次数策略；工具启动认证配置持久化；防火墙开关认证业务；系统密码修改页面实现。

身份鉴别页面的保存主链路只负责口令策略下发：

```text
IdentityPage
  -> IdentitySettingsViewModel.requestSave()
  -> IdentityService.setPasswordPolicyWithResult(config)
  -> securityManager.setPasswordPolicy(ADMIN_WANT, policy)
```

`AuthService` 和 `UserAuthAdapter` 位于 `services/identity/auth`，但它们是共享的系统本地认证基础设施，不是身份鉴别页面保存口令策略的必经链路。

## 2. 状态与数据流 (Data Flow & State)
> **设计原则**: 单向数据流 (View -> ViewModel -> Service -> System API)

- **核心业务状态 (Core Business State)**:
  - `IdentitySettingsState.initialized: boolean`: 页面状态是否完成初始化，默认 `false`。
  - `IdentitySettingsState.loading: boolean`: 当前是否处于初始化读取阶段，默认 `false`。
  - `IdentitySettingsState.adminActivated: boolean`: 企业管理员是否已激活，决定是否允许读取和保存真实系统策略，默认 `false`。
  - `IdentitySettingsState.adminMessage: string`: 管理员状态探测失败时的提示文案，默认空字符串。
  - `IdentitySettingsState.minLengthIndex: number`: 最小密码长度选项索引，映射 `PASSWORD_LENGTH_OPTIONS`，默认 `0`。
  - `IdentitySettingsState.requireUppercase: boolean`: 是否要求大写字母，默认 `false`。
  - `IdentitySettingsState.requireLowercase: boolean`: 是否要求小写字母，默认 `false`。
  - `IdentitySettingsState.requireDigit: boolean`: 是否要求数字，默认 `false`。
  - `IdentitySettingsState.requireSpecialChar: boolean`: 是否要求特殊字符，默认 `false`。
  - `IdentitySettingsState.validityPeriodIndex: number`: 密码有效期选项索引，`0` 半年、`1` 一年、`2` 永久、`3` 自定义，默认 `2`。
  - `IdentitySettingsState.customValidityPeriodText: string`: 自定义有效期输入文本，仅保留数字字符，默认空字符串。
  - `IdentitySettingsState.hasChanges: boolean`: 当前编辑态是否不同于最近一次初始化或保存成功后的基线态，默认 `false`。
  - `IdentitySettingsState.canSave: boolean`: 是否允许点击保存，条件为管理员已激活、非 loading、存在未保存变更，默认 `false`。
- **关键流转路径**:
- 首次进入身份鉴别页面 -> `MainPage` 注入共享 `IdentitySettingsViewModel` -> `IdentityPage.initializeViewModel()` -> `IdentitySettingsViewModel.initialize()` -> `EnterpriseAdminService.probeActivation()` -> `IdentityService.getPasswordPolicyWithResult()` -> `state + initialState` 同步。
- 已初始化后从其它路由返回身份鉴别页面 -> 复用共享 `IdentitySettingsViewModel.state` -> 保留上次退出页面时的当前编辑态与 `hasChanges / canSave`，不重复读取系统策略覆盖未保存变更。
  - 管理员未激活 -> `initialize()` 探测失败 -> `reset()` -> 默认编辑态 -> `canSave = false`。
  - 用户修改复杂度或有效期 -> `setMinLengthIndex()` / `setRequirement()` / `setValidityPeriodIndex()` / `setCustomValidityPeriodText()` -> `refreshDerivedState()` -> `hasChanges / canSave` 更新。
  - 点击保存 -> `requestSave()` -> 重新探测管理员状态 -> `buildConfig()` -> 必要时返回 `confirm` -> `executeSave()` -> `IdentityService.setPasswordPolicyWithResult(config)` -> 结果弹窗。
  - 其它模块需要 PIN / 指纹认证 -> `AuthService.checkAuthMethodAvailability()` 或 `AuthService.authenticate()` -> `UserAuthAdapter` -> `@kit.UserAuthenticationKit`。

身份鉴别模块没有本地持久化仓库。保存后的口令策略不写入 preferences、RDB、JSON 或应用本地文件，而是写入系统 MDM `PasswordPolicy`。页面编辑态由 `ApplicationRuntimeManager` 持有的共享 `IdentitySettingsViewModel` 在主页面生命周期内保留；首次进入时模块通过 `securityManager.getPasswordPolicy(ADMIN_WANT)` 读取系统策略，再解析回显到 `IdentitySettingsState`。路由离开后再进入时优先保留上次退出页面时的编辑态，避免未保存修改被初始化流程覆盖。

#### 3.2 Service / Domain (领域业务层)
* **真实文件路径**:
  * `entry/src/main/ets/services/identity/settings/IdentityService.ets`
  * `entry/src/main/ets/services/identity/settings/IdentityPasswordPolicyMapper.ets`
  * `entry/src/main/ets/services/identity/auth/AuthService.ets`
  * `entry/src/main/ets/services/identity/auth/UserAuthAdapter.ets`
* **核心业务用例 (Use Cases)**:
  * `IdentityPasswordPolicyMapper.toConfig(policy)`: 将系统 `securityManager.PasswordPolicy` 转为页面可编辑的 `PasswordComplexityConfig`。
    * **副作用**: 无。
    * **失败策略**: `policy` 为空时返回 `null`；无法解析最小长度时使用 `0`；通过已知 lookahead 片段识别大小写、数字和特殊字符要求。
  * `IdentityPasswordPolicyMapper.toPasswordPolicy(config)`: 将页面配置转为系统 `PasswordPolicy`。
    * **副作用**: 无。
    * **失败策略**: `validityPeriod` 为数字时转换为毫秒；否则不写入 `validityPeriod` 字段。
  * `IdentityPasswordPolicyMapper.buildComplexityRegex(config)`: 构建系统口令复杂度正则。
    * **副作用**: 无。
    * **失败策略**: 未开启任何字符类型要求时使用 `^.{minLength,}$`；开启要求时追加对应 lookahead 并构建字符集。
  * `IdentityPasswordPolicyMapper.buildDescription(config)`: 构建系统可读的策略描述。
    * **副作用**: 无。
    * **失败策略**: 按最小长度、大写、小写、数字、特殊字符的稳定顺序拼接说明。
  * `IdentityService.getPasswordPolicyWithResult()`: 读取当前系统口令策略。
    * **副作用**: 调用 `securityManager.getPasswordPolicy(ADMIN_WANT)`。
    * **失败策略**: 捕获异常并通过 `OperationResultUtils.buildFailureResult(...)` 包装为 `OperationResult<PasswordPolicy | null>`，失败时 `data` 兜底为 `null`。
  * `IdentityService.setPasswordPolicyWithResult(config)`: 下发系统口令策略。
    * **副作用**: 调用 `securityManager.setPasswordPolicy(ADMIN_WANT, policy)`。
    * **失败策略**: 捕获异常并包装为 `OperationResult<void>`；管理员缺失由统一 failure result 携带 `isAdminMissing`。
  * `AuthService.checkAuthMethodAvailability(authMethod)`: 检查 PIN / 指纹当前是否可用。
    * **副作用**: 调用 `UserAuthAdapter.getAvailableStatus(...)`，间接访问 `userAuth.getAvailableStatus(authType, ATL1)`。
    * **失败策略**: 不支持的认证方式返回 `METHOD_UNSUPPORTED`；系统错误码 `12500010 / 12500006 / 12500013 / 201 / 401` 映射为项目文案。
  * `AuthService.authenticate(authMethod, challenge?)`: 拉起系统认证并返回认证执行结果。
    * **副作用**: 无有效 challenge 时通过 `cryptoFramework.createRandom().generateRandomSync(8)` 生成随机 challenge；创建 `userAuth.UserAuthInstance`，注册 `result` 监听并调用 `start()` 拉起系统认证 UI。
    * **失败策略**: 不支持、challenge 生成失败或系统不可用返回 `reason='unavailable'`；认证失败、取消或非可用性错误返回 `reason='failed'`；成功或回调结束后尝试主动 `off('result')`。
  * `AuthService.getAuthMethodLabel(method)`: 获取认证方式展示名称。
    * **副作用**: 无。
    * **失败策略**: 未知枚举返回通用“认证方式”文本。
  * `UserAuthAdapter.getAvailableStatus(authType)`: 封装 `userAuth.getAvailableStatus(authType, ATL1)`。
    * **副作用**: 调用系统认证可用性 API。
    * **失败策略**: 捕获 `BusinessError`，返回结构化 `UserAuthStatusResult`。
  * `UserAuthAdapter.getUserAuthInstance(request)`: 封装 `userAuth.getUserAuthInstance(authParam, widgetParam)`。
    * **副作用**: 创建系统认证实例。
    * **失败策略**: 捕获 `BusinessError`，返回结构化 `UserAuthInstanceResult`。
* **内部架构设计**:
  * **Repository**: 无。身份鉴别页面不维护本地持久化仓库，口令策略真源是系统 MDM `PasswordPolicy`。
  * **Mapper/Ports**: `IdentityPasswordPolicyMapper` 承担 `PasswordComplexityConfig` 与 `PasswordPolicy` 的 mapper；`UserAuthAdapter` 是系统认证 API port。

`IdentityService`、`IdentityPasswordPolicyMapper`、`AuthService`、`UserAuthAdapter` 的职责边界必须保持清晰：

- `IdentityService` 管“口令策略读写”：只封装 `securityManager.getPasswordPolicy` / `setPasswordPolicy` 调用、taskpool 兜底和异常结果包装。
- `IdentityPasswordPolicyMapper` 管“口令策略如何转换”：最少几位、是否必须包含字符类型、密码多久过期，以及这些配置如何转换为 MDM `PasswordPolicy`。
- `AuthService` 管“是否能证明当前操作者本人”：PIN / 指纹是否可用、如何发起认证、系统认证错误如何映射为业务结果。
- `UserAuthAdapter` 管“如何调用系统认证 API”：只做 `UserAuthenticationKit` 调用与异常结构化，不承载业务文案和业务判断。

`AuthService` 当前映射的认证错误码如下：

| 错误码 | 语义 | 处理结果 |
|---|---|---|
| `12500010` | 未录入或凭据不满足要求 | 返回未录入提示 |
| `12500006` | 认证等级不足 | 返回认证等级不足提示 |
| `12500013` | PIN 过期 | 返回 PIN 过期提示 |
| `201` | 权限不足 | 返回认证权限不足提示 |
| `401` | 参数错误 | 返回认证参数错误提示 |
| 其它 | 未知系统错误 | 优先使用系统 fallback message，否则返回统一不可用提示 |

#### 3.3 ViewModel (视图模型层)
* **真实文件路径**: `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`
* **状态分发逻辑**:
  * `initialize()`: 设置 loading，刷新管理员状态；管理员未激活时重置为默认策略，管理员已激活时通过 `IdentityService.getPasswordPolicyWithResult()` 读取系统策略并回填页面。该方法由页面在共享 ViewModel 未初始化时调用，避免路由返回时覆盖当前编辑态。
  * `reset()`: 调用 `applyPolicy(null)` 回到默认页面编辑态。
  * `setMinLengthIndex(index)`: 当 index 落在 `PASSWORD_LENGTH_OPTIONS` 范围内时更新最小长度索引，并刷新 `hasChanges / canSave`。
  * `setRequirement(key, value)`: 更新大写、小写、数字或特殊字符开关，并刷新派生状态。
  * `setValidityPeriodIndex(index)`: 更新有效期选项；切离自定义时清空 `customValidityPeriodText`。
  * `setCustomValidityPeriodText(value)`: 自定义有效期只保留数字字符。
  * `requestSave(skipImmediateExpiryConfirm=false)`: 保存入口。先重新探测管理员状态，再构建配置；自定义有效期非法时返回失败；自定义 `0` 天且未跳过确认时返回 `confirm`；否则进入 `executeSave(config)`。
  * `executeSave(config)`: 重新读取当前系统策略，与目标配置比较；无变化返回失败提示；有变化则调用 `IdentityService.setPasswordPolicyWithResult(config)`，成功后更新 `initialState` 并清除 dirty 状态。
  * `applyConfig(state, config, policy?)`: 将 `PasswordComplexityConfig` 和原始 `PasswordPolicy.validityPeriod` 映射为页面状态。半年、一年、永久通过毫秒天数候选集合识别，其它正数回填自定义天数。

ViewModel 内部有两个状态基线：`initialState` 是最近一次初始化或保存成功后的快照，`state` 是当前编辑态。`hasChanges` 只比较这两者，不直接依赖页面控件。

口令策略回显链路如下：

```text
IdentityPage.aboutToAppear()
  -> IdentityPage.initializeViewModel()
  -> 若共享 ViewModel 未初始化，则调用 IdentitySettingsViewModel.initialize()
  -> EnterpriseAdminService.probeActivation()
  -> IdentityService.getPasswordPolicyWithResult()
  -> securityManager.getPasswordPolicy(ADMIN_WANT)
  -> IdentityPasswordPolicyMapper.toConfig(policy)
  -> IdentitySettingsViewModel.applyPolicy(policy)
  -> 页面控件读取 viewModel.state 回显
```

回显异常策略：

- 企业管理员未激活：不读取系统策略，回退默认状态。
- 读取策略失败：回退默认状态。
- 系统返回空策略：`IdentityPasswordPolicyMapper.toConfig(null)` 返回 `null`，ViewModel 使用默认状态。
- “永久”不是本地标记，而是当前实现按约 100 年有效期下发，回显时通过毫秒天数候选集合识别。

#### 3.4 View / Page (页面视图层)
* **真实文件路径**: `entry/src/main/ets/views/identity/settings/IdentityPage.ets`
* **结构说明**:
* `IdentityPage`: 身份鉴别页面组合层。接收 `MainPage` 持有的共享 `IdentitySettingsViewModel`；`aboutToAppear()` 仅在共享 ViewModel 未初始化时触发初始化，已初始化时直接复用上次退出页面时的编辑态。页面标题使用 `ModuleText.IDENTITY`；副标题来自 `IdentityPageStrings.SUBTITLE`。
  * 页面主体只有一个 `SectionCard`：`口令复杂度策略`。
  * `viewModel.state.initialized === false` 时，`SectionCard` 内显示轻量加载态，不创建口令策略表单控件；初始化完成后再渲染真实表单，避免 `Select` 先以默认策略创建后再异步回显导致可见文案滞留。
  * 页面字段包括 `最小密码长度`、`要求包含大写字母`、`要求包含小写字母`、`要求包含数字`、`要求包含特殊字符`、`密码有效期`。
  * 保存按钮通过 `IconTextActionButton` 渲染，启用状态绑定 `viewModel.state.canSave`。
  * 保存后页面只消费 `IdentitySettingsActionRequest`：`confirm` 调 `DialogService.openConfirm`，`result` 调 `DialogService.openResult`，`none` 不动作。

  * `IconTextActionButton`: 保存按钮。**入参/事件**: 接收 label、icon、kind、isEnabled，抛出 `onTap()`。

## 4. 异常处理与系统依赖 (Dependencies & Errors)
- **关键系统 API**:
  - `@kit.MDMKit.securityManager.getPasswordPolicy`: 读取系统口令策略。
  - `@kit.MDMKit.securityManager.setPasswordPolicy`: 下发系统口令策略。
  - `@ohos.enterprise.restrictions.getDisallowedPolicy`: `EnterpriseAdminService.probeActivation()` 用于探测企业管理员激活状态。
  - `@kit.UserAuthenticationKit.userAuth.getAvailableStatus`: 检查 PIN / 指纹认证能力是否可用。
  - `@kit.UserAuthenticationKit.userAuth.getUserAuthInstance`: 创建系统认证实例。
  - `@kit.CryptoArchitectureKit.cryptoFramework.createRandom`: 无外部 challenge 时生成认证 challenge。
- **系统权限**:
  - `ohos.permission.ENTERPRISE_MANAGE_SECURITY`
  - `ohos.permission.ENTERPRISE_SET_ACCOUNT_POLICY`
  - `ohos.permission.ENTERPRISE_ADMIN_MANAGE`
  - `ohos.permission.ACCESS_BIOMETRIC`
- **异常兜底策略**:
  - 企业管理员未激活时，`initialize()` 不读取系统口令策略，页面回退默认编辑态；保存时返回失败结果，提示先激活企业管理员。
  - 读取口令策略失败时，ViewModel 记录告警并回退默认编辑态，不阻塞页面显示。
  - 自定义有效期为空或非数字时，`requestSave()` 返回失败提示；页面不调用系统保存。
  - 自定义有效期为 `0` 时，首次保存返回 `confirm`，用户确认后才继续保存。
  - 保存前重新读取当前系统策略并与目标配置比较；无变化时返回“无变更”，不调用 `setPasswordPolicy`。
  - `setPasswordPolicy` 失败时，管理员缺失失败优先显示管理员未激活类提示，其它失败显示重试提示。
  - PIN / 指纹认证不可用时，`AuthService` 将系统错误码映射为业务文案；认证失败或取消返回 `reason='failed'`，系统能力缺失返回 `reason='unavailable'`。
  - 随机 challenge 生成失败时，`AuthService.authenticate()` 返回认证能力不可用，不继续创建认证实例。

### 4.1 实施步骤与测试验收 (Implementation & Acceptance)

- **实施步骤**:
  - 涉及口令策略、复杂度映射、认证能力、企业管理员激活判断、页面交互或权限变化时，先更新本文档的目标范围、状态模型、职责边界、异常兜底和验收口径。
  - 口令配置模型、文案和映射规则先同步 `IdentityModels.ets`、`IdentityStrings.ets`、`IdentityPasswordPolicyMapper.ets`，再调整系统读写服务。
  - 系统口令策略读写统一经 `IdentityService.ets`，认证能力统一经 `AuthService.ets`；页面和 ViewModel 不直接调用 MDM 或 userAuth API。
  - 自定义有效期、无变更保存、管理员未激活和认证不可用等用户可见流程变化，必须同步页面提示、ViewModel 结果类型和测试用例。
  - 新增或调整企业管理/生物认证权限时，同步 `entry/src/main/module.json5`、`hapsigner/UnsgnedDebugProfileTemplate.json` 并按签名流程重新生成 p7b。
- **测试覆盖**:
  - UT: `entry/src/test/identity/*.test.ets` 覆盖口令策略映射、Service、ViewModel 和页面状态；`entry/src/test/auth/*.test.ets` 覆盖 PIN / 指纹认证能力映射。
  - ohosTest: `entry/src/ohosTest/ets/test/simple/RouteAction.test.ets` 通过 `route_action` 场景覆盖身份鉴别入口和页面可达性。
  - E2E: `scripts/e2e/cases/identity/validity_section.json`、`password_policy.json`、`complexity_requirements.json`、`domain_policy.json` 覆盖有效期、口令策略、复杂度和域策略展示。
- **验收口径**:
  - 企业管理员未激活时页面可显示默认编辑态，但保存必须失败并提示激活管理员。
  - 自定义有效期为空或非数字时不调用系统保存；自定义有效期为 `0` 时必须先二次确认。
  - 保存前应比较当前系统策略和目标配置，无变化不调用 `setPasswordPolicy`；认证能力不可用时返回明确业务文案。
