## 1. 业务概述与对外接口 (Overview & Public Interfaces)

- **核心目标**: 提供企业管理员场景下的系统口令复杂度与密码有效期策略配置能力，沉淀可被其它敏感操作复用的 PIN / 指纹本地认证能力。
- **入口路由 (Route Entry)**: `RouteIds.IDENTITY = 'identity'` -> `entry/src/main/ets/views/identity/settings/IdentityPage.ets`
- **对外暴露能力 (Public APIs)**:
  - `IdentityService`: 口令策略读取、保存。
  - `IdentityPasswordPolicyMapper`: `PasswordComplexityConfig` 与系统 `PasswordPolicy` 的双向转换、复杂度正则构建和策略描述构建。
  - `AuthService`: PIN / 指纹可用性检查与系统认证执行。
  - `UserAuthAdapter`: `@kit.UserAuthenticationKit` 的薄适配层，供 `AuthService` 调用并便于测试。
  - `IdentitySettingsViewModel`: 身份鉴别页面编辑态、保存动作和企业管理员状态。
- **业务边界**:
  - ✅ **包含**: 最小密码长度、大小写字母、数字、特殊字符、密码有效期策略配置；企业管理员状态前置感知；通过 `IdentityPasswordPolicyMapper` 完成口令策略与 MDM `PasswordPolicy` 的双向转换；PIN / 指纹认证基础能力封装。
  - ❌ **不包含**: 企业管理员激活流程本身；账号体系、统一身份平台、账号锁定/失败次数策略；工具启动认证配置持久化；防火墙开关认证业务；系统密码修改页面实现；SecurityTool 页面内读取 UKey、保存 UKey 绑定、注入或删除 CustomAuth 凭据、声明 CustomAuthenticator appService、实现系统锁屏页/SystemUI 解锁流程。

身份鉴别页面的保存主链路只负责口令策略下发：

```text
IdentityPage
  -> IdentitySettingsViewModel.requestSave()
  -> IdentityService.setPasswordPolicyWithResult(config)
  -> securityManager.setPasswordPolicy(ADMIN_WANT, policy)
```

`AuthService` 和 `UserAuthAdapter` 位于 `services/identity/auth`，但它们是共享的系统本地认证基础设施，不是身份鉴别页面保存口令策略的必经链路。

UKey 认证能力已从 SecurityTool 迁出。SecurityTool 不再显示 `UKey 锁屏认证` 开关，不再保存 UKey 配置，不再响应 USB attach / detach 做凭据注册或删除，也不再内置 `ICustomAuthenticatorV1` appService。相关能力由 `ukey/` 独立系统应用按系统签名承载。

## 2. 状态与数据流 (Data Flow & State)

> **设计原则**: 单向数据流 (View -> ViewModel -> Service -> System API)

- **核心业务状态 (Core Business State)**:
  - `IdentitySettingsState.initialized: boolean`: 页面状态是否完成初始化，默认 `false`。
  - `IdentitySettingsState.loading: boolean`: 当前是否处于初始化读取阶段，默认 `false`。
  - `IdentitySettingsState.adminActivated: boolean`: 企业管理员是否已激活，决定是否允许读取和保存真实系统策略，默认 `false`。
  - `IdentitySettingsState.adminMessage: string`: 管理员状态探测失败时的提示文案，默认空字符串。
  - `IdentitySettingsState.minLengthIndex: number`: 最小密码长度选项索引，映射 `PASSWORD_LENGTH_OPTIONS`，默认 `0`。
  - `IdentitySettingsState.requireUppercase / requireLowercase / requireDigit / requireSpecialChar: boolean`: 复杂度要求开关。
  - `IdentitySettingsState.validityPeriodIndex: number`: 密码有效期选项索引，`0` 半年、`1` 一年、`2` 永久、`3` 自定义，默认 `2`。
  - `IdentitySettingsState.customValidityPeriodText: string`: 自定义有效期输入文本，仅保留数字字符，默认空字符串。
  - `IdentitySettingsState.hasChanges: boolean`: 当前编辑态是否不同于最近一次初始化或保存成功后的基线态，默认 `false`。
  - `IdentitySettingsState.canSave: boolean`: 是否允许点击保存，条件为管理员已激活、非 loading、存在未保存变更，默认 `false`。
- **关键流转路径**:
  - 首次进入身份鉴别页面 -> `MainPage` 注入共享 `IdentitySettingsViewModel` -> `IdentityPage.initializeViewModel()` -> `IdentitySettingsViewModel.initialize()` -> `EnterpriseAdminService.probeActivation()` -> `IdentityService.getPasswordPolicyWithResult()` -> `state + initialState` 同步。
  - 已初始化后从其它路由返回身份鉴别页面 -> 复用共享 `IdentitySettingsViewModel.state` -> 保留上次退出页面时的当前编辑态与 `hasChanges / canSave`，不重复读取系统策略覆盖未保存变更。
  - 管理员未激活 -> `initialize()` 探测失败 -> `reset()` -> 默认编辑态 -> `canSave = false`。
  - 用户修改复杂度或有效期 -> `setMinLengthIndex()` / `setRequirement()` / `setValidityPeriodIndex()` / `setCustomValidityPeriodText()` -> `refreshDerivedState()` -> `hasChanges / canSave` 更新。
  - 点击保存 -> `requestSave()` -> 重新探测管理员状态 -> `buildConfig()` -> 必要时返回 `confirm` -> `executeSave()` -> `IdentityService.setPasswordPolicyWithResult(config)` -> 结果弹窗。
  - 其它模块需要 PIN / 指纹认证 -> `AuthService.checkAuthMethodAvailability()` 或 `AuthService.authenticate()` -> `UserAuthAdapter` -> `@kit.UserAuthenticationKit`。

口令策略没有本地持久化仓库。保存后的口令策略不写入 preferences、RDB、JSON 或应用本地文件，而是写入系统 MDM `PasswordPolicy`。页面编辑态由 `ApplicationRuntimeManager` 持有的共享 `IdentitySettingsViewModel` 在主页面生命周期内保留；首次进入时模块通过 `securityManager.getPasswordPolicy(ADMIN_WANT)` 读取系统策略，再解析回显到 `IdentitySettingsState`。路由离开后再进入时优先保留上次退出页面时的编辑态，避免未保存修改被初始化流程覆盖。

## 3. 核心功能场景 (Core Functional Scenarios)

- **口令复杂度配置**: 支持最小密码长度、大小写字母、数字、特殊字符要求。
- **密码有效期配置**: 支持半年、一年、永久和自定义天数；自定义 `0` 天表示马上过期，需要二次确认。
- **企业管理员状态感知**: 未激活时页面可展示默认编辑态，但保存必须失败并提示先激活企业管理员。
- **本地认证基础能力**: 提供 PIN / 指纹可用性检查和认证执行封装，供工具设置、防火墙等敏感操作复用。
- **UKey 能力迁出**: 身份鉴别页面不再展示锁屏 UKey 认证区域；UKey 管理、凭据生命周期和认证器执行端由 `ukey/` 独立系统应用承载。

## 4. 模块结构与组件设计 (Module Components)

### 【核心层】(Core MVVM & Domain Layers)

#### 4.1 Model & Types

- `PasswordComplexityConfig`
  - **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
  - **业务作用**: 页面编辑态和系统 `PasswordPolicy` 之间的口令复杂度中间模型。
- `PASSWORD_LENGTH_OPTIONS`
  - **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
  - **业务作用**: 身份鉴别页面最小密码长度下拉选项。
- `PASSWORD_VALIDITY_PRESETS` / `PASSWORD_VALIDITY_DAY_IN_MS`
  - **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
  - **业务作用**: 有效期选项、毫秒换算和默认语义。
- `AuthMethod`
  - **所在文件**: `entry/src/main/ets/models/AuthModels.ets`
  - **业务作用**: 项目内部认证方式枚举，当前包含 PIN 和指纹。
- `IdentitySettingsState`
  - **所在文件**: `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`
  - **业务作用**: 身份鉴别页面编辑态和派生保存状态。
- `IdentitySettingsActionRequest`
  - **所在文件**: `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`
  - **业务作用**: ViewModel 向页面返回保存动作请求，避免页面自行判断业务分支。

#### 4.2 ViewModel

- `IdentitySettingsViewModel`
  - 初始化时探测企业管理员状态并读取系统口令策略。
  - 维护页面编辑态、未保存变更、保存可用状态和二次确认动作。
  - 保存前重新探测管理员状态并比较当前系统策略，避免无变化时重复下发。

#### 4.3 Service / Adapter

- `IdentityService`
  - 封装 `securityManager.getPasswordPolicy` / `setPasswordPolicy`。
  - 将系统异常映射为 `OperationResult`。
- `IdentityPasswordPolicyMapper`
  - 负责页面模型和系统 `PasswordPolicy` 双向转换。
  - 负责复杂度正则和规则描述生成。
- `AuthService`
  - 封装 PIN / 指纹认证可用性检查和认证执行。
  - 将系统认证错误码映射为业务文案。
- `UserAuthAdapter`
  - 对 `@kit.UserAuthenticationKit` 做薄封装，便于测试替换。

#### 4.4 View

- `IdentityPage`
  - 显示口令复杂度策略卡片。
  - 头部保存按钮只保存口令策略，不处理 UKey。
  - 不展示锁屏 UKey 认证卡片、验证按钮、绑定状态或测试入口。

#### 4.5 Ability / Runtime

- `entry/src/main/ets/pages/MainPage.ets`
  - 渲染 `IdentityPage`，当 `currentPage === RouteIds.IDENTITY` 时展示身份鉴别页面。
  - 从 `ApplicationRuntimeManager` 注入共享 `IdentitySettingsViewModel`。
- `entry/src/main/ets/runtime/ApplicationRuntimeManager.ets`
  - 持有共享 `IdentitySettingsViewModel`。
  - 身份鉴别运行时不再挂载 UKey consumer 或启动 UKey 对账。

## 5. 异常处理与系统依赖 (Dependencies & Errors)

- **关键系统 API**:
  - `@kit.MDMKit.securityManager.getPasswordPolicy`
  - `@kit.MDMKit.securityManager.setPasswordPolicy`
  - `@ohos.enterprise.restrictions.getDisallowedPolicy`
  - `@kit.UserAuthenticationKit.userAuth.getAvailableStatus`
  - `@kit.UserAuthenticationKit.userAuth.getUserAuthInstance`
- **系统权限**:
  - `ohos.permission.ENTERPRISE_MANAGE_SECURITY`
  - `ohos.permission.ENTERPRISE_SET_ACCOUNT_POLICY`
  - `ohos.permission.ENTERPRISE_ADMIN_MANAGE`
  - `ohos.permission.ACCESS_BIOMETRIC`
- **不再由 SecurityTool 身份鉴别模块申请的 UKey 权限**:
  - `ohos.permission.MANAGE_USER_IDM`
  - `ohos.permission.USE_USER_IDM`
  - `ohos.permission.ACCESS_USER_AUTH_INTERNAL`
  - `ohos.permission.ACCESS_PIN_AUTH`
  - `ohos.permission.ACCESS_CUSTOM_AUTHENTICATOR`
  - `ohos.permission.ACCESS_EXTENSIONAL_DEVICE_DRIVER`
- **异常兜底策略**:
  - 企业管理员未激活时，`initialize()` 不读取系统口令策略，页面回退默认编辑态；保存时返回失败结果，提示先激活企业管理员。
  - 读取口令策略失败时，ViewModel 记录告警并回退默认编辑态，不阻塞页面显示。
  - 系统未配置口令策略时回显为默认“永久”；默认“永久”不是 `0`；系统空策略携带 `validityPeriod = 0` 仍按未配置处理。
  - 非空系统策略或保存目标中的 `validityPeriod = 0` 回显为自定义 `0` 天，表示马上过期。
  - 自定义有效期为空或非数字时，`requestSave()` 返回失败提示；页面不调用系统保存。
  - 自定义有效期为 `0` 时表示马上过期，首次保存返回 `confirm`，用户确认后才继续保存。
  - 保存前重新读取当前系统策略并与目标配置比较；无变化时返回“无变更”，不调用 `setPasswordPolicy`。
  - PIN / 指纹认证不可用时，`AuthService` 将系统错误码映射为业务文案；认证失败或取消返回 `reason='failed'`，系统能力缺失返回 `reason='unavailable'`。
  - 随机 challenge 生成失败时，`AuthService.authenticate()` 返回认证能力不可用，不继续创建认证实例。

### 5.1 实施步骤与测试验收

- **实施步骤**:
  - 涉及口令策略、复杂度映射、认证能力、企业管理员激活判断、页面交互或权限变化时，先更新本文档的目标范围、状态模型、职责边界、异常兜底和验收口径。
  - 口令配置模型、文案和映射规则先同步 `IdentityModels.ets`、`IdentityStrings.ets`、`IdentityPasswordPolicyMapper.ets`，再调整系统读写服务。
  - 系统口令策略读写统一经 `IdentityService.ets`，认证能力统一经 `AuthService.ets`；页面和 ViewModel 不直接调用 MDM 或 userAuth API。
  - 新增或调整企业管理/生物认证权限时，同步 `entry/src/main/module.json5`、`hapsigner/UnsgnedDebugProfileTemplate.json` 并按签名流程重新生成 p7b。
- **测试覆盖**:
  - UT: `entry/src/test/identity/*.test.ets` 覆盖口令策略映射、Service、ViewModel 和页面状态。
  - UT: `entry/src/test/auth/*.test.ets` 覆盖 PIN / 指纹认证能力映射。
  - ohosTest: `entry/src/ohosTest/ets/test/simple/RouteAction.test.ets` 通过 `route_action` 场景覆盖身份鉴别入口和页面可达性。
  - E2E: `scripts/e2e/cases/identity/validity_section.json`、`password_policy.json`、`complexity_requirements.json`、`domain_policy.json` 覆盖有效期、口令策略、复杂度和域策略展示。
- **验收口径**:
  - 企业管理员未激活时页面可显示默认编辑态，但保存必须失败并提示激活管理员。
  - 自定义有效期为空或非数字时不调用系统保存。
  - 自定义有效期为 `0` 时必须先二次确认，非空策略回显也必须保留为自定义 `0`。
  - 保存前应比较当前系统策略和目标配置，无变化不调用 `setPasswordPolicy`。
  - 身份鉴别页面不展示 `UKey 锁屏认证` 区域。
  - SecurityTool 不声明 CustomAuthenticator appService，不引入 UKey DDK、User IDM 或 PIN token 凭据链路权限。
