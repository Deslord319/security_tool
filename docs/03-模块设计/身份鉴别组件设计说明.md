# 身份鉴别组件设计说明

## 1. 文档目的

本文档描述 SecurityTool 中“身份鉴别”组件的功能边界、MVVM 分层、核心状态模型、关键数据流和维护要点，作为后续维护、联调和扩展的基线文档。

当前版本中，身份鉴别模块聚焦于企业管理员维度的口令策略配置，核心语义如下：

- 页面负责展示和交互编排
- ViewModel 负责初始化、管理员感知、口令策略状态编辑、差异计算和保存请求
- Service 负责口令策略与系统 MDM PasswordPolicy 的双向转换
- 未激活企业管理员时，页面只能展示配置项，不能真正落策略

---

## 2. 功能范围

身份鉴别组件当前包含一类核心能力：

1. 口令复杂度策略配置

### 2.1 口令复杂度策略配置

当前页面可配置的策略项包括：

- 最小密码长度
- 是否要求包含大写字母
- 是否要求包含小写字母
- 是否要求包含数字
- 是否要求包含特殊字符
- 密码有效期

密码有效期支持两类输入方式：

- 预置选项：半年、一年、永久
- 自定义输入：用户输入非负整数天数，`0` 表示密码立即过期风险

当前实现补充规则：

- 下拉展示顺序为：半年、一年、永久、自定义
- 默认值为“永久”，实现上按 100 年有效期下发
- 当系统侧未读取到有效期，或读取结果小于等于 0 天时，界面按“永久”回显

### 2.2 管理员激活感知

身份鉴别模块的策略下发依赖企业管理员能力。页面会在进入时探测管理员状态：

- 已激活：读取当前系统口令策略并初始化页面
- 未激活：页面回退到默认状态，保存时给出明确失败提示

## 3. 业务语义模型

### 3.1 状态分层

```text
第 1 层：管理员状态
- 决定是否允许真实读取 / 保存系统口令策略

第 2 层：页面编辑态
- 维护当前用户选择的复杂度和有效期配置

第 3 层：系统 PasswordPolicy
- 由 MDMKit securityManager 持有的真实策略
```

### 3.2 页面编辑态语义

ViewModel 内部维护两份编辑态快照，并额外维护派生 UI 状态：

- `initialState`：最近一次成功初始化或保存后的基线状态
- `state`：当前正在编辑的状态
- `state.initialized / loading / saving`：初始化与保存阶段
- `state.adminActivated / adminMessage`：管理员状态感知
- `state.canSave / validationMessage`：派生保存状态与校验结果

`hasChanges` 的语义为：

- 当前编辑态与基线态不同，则为 `true`
- 相同则为 `false`

这样页面无需自己做差异判断，保存入口只需调用 `viewModel.requestSave()`

- 页面进入时触发 `viewModel.initialize()`
- 将用户操作转发给 ViewModel
- 保存时根据 `requestSave()` 返回结果弹出确认框或结果对话框

- 维护身份鉴别编辑态

- 生成待保存的 `PasswordComplexityConfig`
- 对比当前策略和目标策略
- 调用 `IdentityService` 完成落盘
- 输出 `none / confirm / result` 三类动作请求

### 7.4 保存数据流

```mermaid
sequenceDiagram
  participant UI as IdentityPage
  participant VM as IdentitySettingsViewModel
  participant Admin as EnterpriseAdminService
  participant Service as IdentityService
  participant MDM as securityManager
  participant Dialog as DialogService

  UI->>VM: requestSave()
  VM->>Admin: probeActivation()
  Admin-->>VM: AdminActivationState
  alt 管理员未激活
    VM-->>UI: result(failure)
    UI->>Dialog: openResult(failure)
  else 管理员已激活
    VM->>VM: buildConfig()
    alt 需要立即过期确认
      VM-->>UI: confirm
      UI->>Dialog: openConfirm(...)
    else 进入真实保存
      VM->>Service: getPasswordPolicyWithResult()
      Service->>MDM: getPasswordPolicy()
      MDM-->>Service: current policy
      Service-->>VM: current OperationResult
      VM->>VM: compare current config / target config
      alt 没有变化
        VM-->>UI: result(failure)
        UI->>Dialog: openResult(failure)
      else 有变化
        VM->>Service: setPasswordPolicyWithResult(config)
        Service->>MDM: setPasswordPolicy(policy)
        MDM-->>Service: success / error
        Service-->>VM: OperationResult
        VM-->>UI: result(success/failure)
        UI->>Dialog: openResult(success/failure)
      end
    end
  end
```

## 8. 关键交互说明

### 8.1 页面进入

页面进入时总是先做管理员状态探测，再决定是否加载真实策略：

- 已激活：加载系统真实策略
- 未激活：页面重置为默认值

### 8.2 自定义有效期

当用户选择“自定义”时：

- `validityPeriodIndex` 切换为 `CUSTOM_VALIDITY_OPTION_INDEX`
- 输入框显示
- 输入内容只保留数字字符
- 若输入 `0`，页面会提示“密码可能立即过期”的风险确认

### 8.3 保存判定

保存时的判断顺序为：

1. 是否正在保存
2. 是否已激活管理员
3. 自定义有效期是否合法
4. 是否命中“立即过期”确认分支
5. 当前编辑结果与系统真实策略是否有差异
6. 调用 MDM 保存

## 10. 主要验收点

- 管理员激活时进入页面，可正确回显当前口令策略
- 未激活时进入页面，保存会提示管理员未激活
- 切换复杂度项后，`hasChanges` 状态正确变化
- 选择自定义有效期时，非法输入会被拦截
- 自定义输入 `0` 时，会出现立即过期风险提示
- 未修改直接保存会提示“无变更”
- 保存成功后，再次进入页面可回显新策略

## 12. 维护建议

- 页面层只做初始化触发、交互转发和结果提示，不直接探测管理员或拼装正则
- 所有复杂度规则变更统一收口到 `IdentityService.buildComplexityRegex`
- 所有系统 PasswordPolicy 回显逻辑统一收口到 `IdentitySettingsViewModel.initialize`
- 后续新增策略项时，需同时更新：
  - `IdentitySettingsState`
  - `buildConfig` / `applyConfig`
  - `IdentityService.toConfig` / `IdentityService.toPasswordPolicy`
  - `IdentityService.buildComplexityRegex`
- 若修改 `requestSave()`、`IdentitySettingsActionRequest` 或状态字段，必须同步更新：
  - `entry/src/test/identity/settings-viewmodel.test.ets`
  - `entry/src/ohosTest/ets/test/identity/page.test.ets`
  - 手工测试清单中的身份鉴别章节
