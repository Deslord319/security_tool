## 1. 业务概述与对外接口 (Overview & Public Interfaces)
- **核心目标**: 提供企业管理员场景下的系统口令复杂度与密码有效期策略配置能力，沉淀可被其它敏感操作复用的 PIN / 指纹本地认证能力，并提供 UKey 认证能力的独立配置入口。
- **入口路由 (Route Entry)**: `RouteIds.IDENTITY = 'identity'` -> `entry/src/main/ets/views/identity/settings/IdentityPage.ets`
- **对外暴露能力 (Public APIs)**:
  - `IdentityService`: 口令策略读取、保存。
  - `IdentityPasswordPolicyMapper`: `PasswordComplexityConfig` 与系统 `PasswordPolicy` 的双向转换、复杂度正则构建和策略描述构建。
  - `AuthService`: PIN / 指纹可用性检查与系统认证执行。
  - `UserAuthAdapter`: `@kit.UserAuthenticationKit` 的薄适配层，供 `AuthService` 调用并便于测试。
  - `LockScreenAuthRepository`: 锁屏认证本地配置读取与保存，当前包含 UKey 锁屏解锁开关。
  - `LockScreenUKeyBindingRepository`: 锁屏 UKey 首次注册后的可信绑定记录读取与保存，拔出时不删除。
  - `LockScreenUKeyActiveCredentialRepository`: 读取、保存和清空当前系统侧已注入的活动 CustomAuth 凭据。
  - `LockScreenUKeyDeviceService`: 默认通过 DDK `deviceManager.queryDevices(BusType.USB)` 读取当前插入的 UKey / USB 设备管理标识，并用 `usbManager.getDevices()` 补充设备详情。
  - `LockScreenCustomAuthEnrollmentService`: 编排“单 UKey 检测 -> CustomAuth 凭据注册/删除 -> 绑定与活动凭据保存”。
  - `LockScreenUKeyRuntimeEnrollmentConsumer`: 复用外设运行时 USB attach / detach 事件，在锁屏 UKey 开关开启时触发首把 UKey 注册或活动凭据删除。
  - `LockScreenAuthViewModel`: 锁屏认证设置区状态、即时保存和后台注册动作。
- **业务边界**:
  - ✅ **包含**: 最小密码长度、大小写字母、数字、特殊字符、密码有效期策略配置；企业管理员状态前置感知；通过 `IdentityPasswordPolicyMapper` 完成口令策略与 MDM `PasswordPolicy` 的双向转换；PIN / 指纹认证基础能力封装；UKey 认证开关的独立持久化配置与页面入口；当前注册骨架中，开关开启时基于当前唯一插入 USB/UKey 或 USB attach 事件发起 CustomAuth 凭据注册；首次注册成功后保存 MDM 侧 UKey 可信绑定和活动凭据记录，后续其它 USB/UKey 不再注册为新的认证凭据；首把 UKey 拔出后删除活动凭据但保留可信绑定。
  - ❌ **不包含**: 企业管理员激活流程本身；账号体系、统一身份平台、账号锁定/失败次数策略；工具启动认证配置持久化；防火墙开关认证业务；系统密码修改页面实现；SecurityTool 页面内读取 UKey 私钥、PIN、证书私有材料或直接做认证判断；系统锁屏页/SystemUI 解锁流程实现；UKey DDK backend 和 CustomAuthenticator 执行端的完整实现。后两者的目标设计由 `UKey管理与认证器接入设计说明.md` 承接。

身份鉴别页面的保存主链路只负责口令策略下发：

```text
IdentityPage
  -> IdentitySettingsViewModel.requestSave()
  -> IdentityService.setPasswordPolicyWithResult(config)
  -> securityManager.setPasswordPolicy(ADMIN_WANT, policy)
```

`AuthService` 和 `UserAuthAdapter` 位于 `services/identity/auth`，但它们是共享的系统本地认证基础设施，不是身份鉴别页面保存口令策略的必经链路。

UKey 认证能力属于身份鉴别模块的独立配置，不复用工具设置的启动认证配置，不影响 `EntryAbility` 应用启动认证，也不改变防火墙敏感操作 PIN 认证。SecurityTool 侧只提供一个“UKey 锁屏认证”开关，不提供手动验证、验证结果展示、UKey 切换或 UKey 删除入口。当前已落地代码包含单开关、DDK USB 设备发现、首把 USB/UKey 注册骨架、活动凭据持久化、外设 attach 触发注册和 detach 触发删除活动凭据；锁屏解锁能力先作为系统后续调度入口预留，不作为当前重点。CustomAuth 核心逻辑裁剪接入和第二把 UKey 严格失败规则统一收敛到 `UKey管理与认证器接入设计说明.md`。

当前 SecurityTool 的开关、首把 UKey 可信绑定和活动凭据均落在本应用独立 Preferences 中；绑定记录包含 `userId`、`credentialIdHex`、UKey 管理标识、设备描述和注册时间，不包含 UKey 私钥、PIN、证书私有材料或认证结果。活动凭据记录只保存系统凭据 ID、账号、pluginInfo、绑定设备标识和生命周期，用于拔出时删除本应用管理的系统凭据。当前已落地骨架仍按外部测试认证器注册；后续只裁剪复用认证器核心协议和密钥逻辑，DDK 后端接入按 `UKey管理与认证器接入设计说明.md` 的专项设计推进。

## 2. 状态与数据流 (Data Flow & State)
> **设计原则**: 单向数据流 (View -> ViewModel -> Service -> System API)

- **核心业务状态 (Core Business State)**:
  - `IdentitySettingsState.initialized: boolean`: 页面状态是否完成初始化，默认 `false`。
  - `IdentitySettingsState.loading: boolean`: 当前是否处于初始化读取阶段，默认 `false`。
  - `IdentitySettingsState.adminActivated: boolean`: 企业管理员是否已激活，决定是否允许读取和保存真实系统策略，默认 `false`。
  - `IdentitySettingsState.adminMessage: string`: 管理员状态探测失败时的提示文案，默认空字符串。
  - `IdentitySettingsState.minLengthIndex: number`: 最小密码长度选项索引，映射 `PASSWORD_LENGTH_OPTIONS`，默认 `0`。
  - `IdentitySettingsState.requireUppercase: boolean`: 是否要求大写字母，默认 `false`。
  - `IdentitySettingsState.requireLowercase: boolean`: 是否要求小写字母，默认 `false`。
  - `IdentitySettingsState.requireDigit: boolean`: 是否要求数字，默认 `false`。
  - `IdentitySettingsState.requireSpecialChar: boolean`: 是否要求特殊字符，默认 `false`。
  - `IdentitySettingsState.validityPeriodIndex: number`: 密码有效期选项索引，`0` 半年、`1` 一年、`2` 永久、`3` 自定义，默认 `2`。
  - `IdentitySettingsState.customValidityPeriodText: string`: 自定义有效期输入文本，仅保留数字字符，默认空字符串。
  - `IdentitySettingsState.hasChanges: boolean`: 当前编辑态是否不同于最近一次初始化或保存成功后的基线态，默认 `false`。
  - `IdentitySettingsState.canSave: boolean`: 是否允许点击保存，条件为管理员已激活、非 loading、存在未保存变更，默认 `false`。
  - `LockScreenAuthState.initialized: boolean`: 锁屏认证设置是否完成初始化，默认 `false`。
  - `LockScreenAuthState.loading: boolean`: 当前是否正在读取锁屏认证配置，默认 `false`。
  - `LockScreenAuthState.saving: boolean`: 当前是否正在保存锁屏认证配置，默认 `false`。
  - `LockScreenAuthState.enrolling: boolean`: 当前是否正在尝试注册首把 UKey 锁屏认证凭据，默认 `false`。
  - `LockScreenAuthState.ukeyUnlockEnabled: boolean`: 是否启用锁屏 UKey 认证入口，默认 `true`。
  - `LockScreenAuthState.ukeyCredentialRegistered: boolean`: MDM 侧是否已有首把 UKey 绑定记录，默认 `false`。
  - `LockScreenAuthState.message: string`: 最近一次读取或保存失败提示，默认空字符串。
- **关键流转路径**:
- 首次进入身份鉴别页面 -> `MainPage` 注入共享 `IdentitySettingsViewModel` -> `IdentityPage.initializeViewModel()` -> `IdentitySettingsViewModel.initialize()` -> `EnterpriseAdminService.probeActivation()` -> `IdentityService.getPasswordPolicyWithResult()` -> `state + initialState` 同步。
- 已初始化后从其它路由返回身份鉴别页面 -> 复用共享 `IdentitySettingsViewModel.state` -> 保留上次退出页面时的当前编辑态与 `hasChanges / canSave`，不重复读取系统策略覆盖未保存变更。
  - 管理员未激活 -> `initialize()` 探测失败 -> `reset()` -> 默认编辑态 -> `canSave = false`。
  - 用户修改复杂度或有效期 -> `setMinLengthIndex()` / `setRequirement()` / `setValidityPeriodIndex()` / `setCustomValidityPeriodText()` -> `refreshDerivedState()` -> `hasChanges / canSave` 更新。
  - 点击保存 -> `requestSave()` -> 重新探测管理员状态 -> `buildConfig()` -> 必要时返回 `confirm` -> `executeSave()` -> `IdentityService.setPasswordPolicyWithResult(config)` -> 结果弹窗。
  - 首次进入身份鉴别页面 -> `MainPage` 注入共享 `LockScreenAuthViewModel` -> `IdentityPage.initializeLockScreenAuthViewModel()` -> `LockScreenAuthViewModel.initialize(context)` -> `LockScreenAuthRepository.loadConfig(context)` + `LockScreenUKeyBindingRepository.loadBinding(context)` -> 回填 `ukeyUnlockEnabled / ukeyCredentialRegistered`；若开关开启且未存在绑定，则后台调用 `LockScreenCustomAuthEnrollmentService.ensureFirstUKeyRegistered(context)` 尝试注册首把 UKey。
  - 切换锁屏 UKey 解锁开关 -> `LockScreenAuthViewModel.setUKeyUnlockEnabled(context, value)` -> `LockScreenAuthRepository.saveConfig(context, config)` -> 成功后更新状态；若切换为开启则尝试首把 UKey 注册；保存失败时回退到原值并返回失败动作。
  - 首把 UKey 注册 -> `LockScreenCustomAuthEnrollmentService` 读取当前 USB 候选 -> 0 把或多把直接失败，不调用注册接口；1 把且无历史绑定时调用 CustomAuth 凭据注册；注册成功后保存绑定记录；已有绑定时不为后续插入的其它设备重新注册。
  - USB 插入/拔出事件 -> `ApplicationRuntimeManager` 已启动外设运行时 -> `PeripheralConnectionRecordRuntimeProducerAdapter` -> `PeripheralConnectionRecordPipelineImpl` -> `LockScreenUKeyRuntimeEnrollmentConsumer` -> 读取锁屏 UKey 开关；开关开启时 attach 调用 `LockScreenCustomAuthEnrollmentService.onUKeyAttached(context)`，detach 调用 `LockScreenCustomAuthEnrollmentService.onUKeyDetached(context)`；该 Consumer 返回 `null`，不生成外设连接记录、不影响外设黑白名单语义。
  - 其它模块需要 PIN / 指纹认证 -> `AuthService.checkAuthMethodAvailability()` 或 `AuthService.authenticate()` -> `UserAuthAdapter` -> `@kit.UserAuthenticationKit`。

口令策略没有本地持久化仓库。保存后的口令策略不写入 preferences、RDB、JSON 或应用本地文件，而是写入系统 MDM `PasswordPolicy`。锁屏 UKey 认证开关和首把 UKey 绑定使用独立 Preferences store `lockscreen_auth_settings` 持久化，不写入 RDB，也不与工具设置 `tool_settings` 或外设管理 `peripheral_device_policy` 混用。页面编辑态由 `ApplicationRuntimeManager` 持有的共享 `IdentitySettingsViewModel` 和 `LockScreenAuthViewModel` 在主页面生命周期内保留；首次进入时模块通过 `securityManager.getPasswordPolicy(ADMIN_WANT)` 读取系统策略，再解析回显到 `IdentitySettingsState`，同时通过 Preferences 读取锁屏认证配置和首把 UKey 绑定。路由离开后再进入时优先保留上次退出页面时的编辑态，避免未保存修改被初始化流程覆盖。

## 3. 核心功能场景 (Core Functional Scenarios)

- **锁屏 UKey 认证默认开启**: 新安装、历史无配置或读取失败时，身份鉴别页“锁屏认证”区域默认显示 UKey 锁屏认证开启；用户关闭后重启应用或设备仍保持关闭。
- **单开关页面**: 锁屏认证区域只显示一个 `UKey 锁屏认证` 开关；不显示验证按钮、验证结果、UKey 绑定、UKey 切换或 UKey 删除入口。
- **首把 UKey 绑定规则**: 开关开启且 MDM 侧尚无绑定时，只有当前恰好插入一把候选 USB/UKey 才发起注册；注册成功后该设备成为首把绑定，后续插入其它 USB/UKey 不触发重新注册或替换。
- **USB 插入/拔出触发凭据生命周期**: 应用运行时外设采集已启动后，USB attach 事件会在开关开启时触发一次首把注册尝试；已有绑定、开关关闭、无 UKey 或多 UKey 均不注册。USB detach 事件会在开关开启时检查首把是否仍在场；首把已不在场且存在活动凭据时删除该活动凭据，但保留可信绑定。
- **锁屏能力预留**: 真正锁屏时不由 SecurityTool 页面参与；SystemUI/锁屏页后续调用系统认证链路并调度认证器。本阶段不把真实锁屏解锁作为身份鉴别页验收重点，先验收 UKey 管理、凭据注入/删除和认证器严格匹配前置链路。

## 4. 模块结构与组件设计 (Module Components)

### 【核心层】(Core MVVM & Domain Layers)

#### 4.1 Model & Types (核心数据模型与类型)
* **核心实体与依赖路径**:
  * `PasswordComplexityConfig`
    * **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
    * **业务作用**: 页面编辑态和系统 `PasswordPolicy` 之间的口令复杂度中间模型。
    * **关键字段**: `minLength` (最小密码长度), `requireUppercase / requireLowercase / requireDigit / requireSpecialChar` (复杂度开关), `validityPeriod` (有效期天数，可为 `undefined`；非空策略中的 `0` 表示马上过期；空策略携带 `0` 视为系统未配置；默认“永久”不是 `0`)。
  * `PASSWORD_LENGTH_OPTIONS`
    * **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
    * **业务作用**: 身份鉴别页面最小密码长度下拉选项。
    * **关键字段**: `[6, 8, 10, 12, 16, 20, 24]`。
  * `PASSWORD_VALIDITY_PRESETS`
    * **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
    * **业务作用**: 有效期预置语义，ViewModel 基于该数组长度确定自定义选项索引。
    * **关键字段**: `halfYear`, `oneYear`, `permanent`。
  * `PASSWORD_VALIDITY_DAY_IN_MS` / `PASSWORD_*_VALIDITY_DAY_OPTIONS`
    * **所在文件**: `entry/src/main/ets/models/IdentityModels.ets`
    * **业务作用**: 身份鉴别有效期天数和毫秒换算的统一常量来源，供 `IdentitySettingsViewModel` 与 `IdentityPasswordPolicyMapper` 共享。
    * **关键字段**: `PASSWORD_VALIDITY_DAY_IN_MS`、`PASSWORD_HALF_YEAR_VALIDITY_DAY_OPTIONS`、`PASSWORD_ONE_YEAR_VALIDITY_DAY_OPTIONS`、`PASSWORD_PERMANENT_VALIDITY_DAY_OPTIONS`、`PASSWORD_PERMANENT_VALIDITY_YEARS`。
  * `AdminActivationState`
    * **所在文件**: `entry/src/main/ets/models/CommonModels.ets`
    * **业务作用**: 企业管理员探测结果，控制身份鉴别是否可读写系统策略。
    * **关键字段**: `activated` (是否激活), `errorCode / errorMessage` (失败原因), `enableCommand` (激活命令), `adminType` (管理员类型)。
    * **提示口径**: `EnterpriseAdminService` 同步与异步探测路径都必须把 `ADMIN_NOT_ACTIVATED` 映射为统一未激活提示，避免同一系统码在页面间展示不同文案。
  * `OperationResult<T>`
    * **所在文件**: `entry/src/main/ets/models/CommonModels.ets`
    * **业务作用**: Service 层统一返回结构。
    * **关键字段**: `success` (是否成功), `data` (成功数据), `errorCode / errorMessage` (系统错误), `isAdminMissing` (管理员缺失标记)。
  * `AuthMethod`
    * **所在文件**: `entry/src/main/ets/models/AuthModels.ets`
    * **业务作用**: 项目内部认证方式枚举。
    * **关键字段**: `PIN = 1`, `FINGERPRINT = 4`。
  * `LockScreenAuthConfig`
    * **所在文件**: `entry/src/main/ets/models/identity/lockscreen-auth/LockScreenAuthModels.ets`
    * **业务作用**: 锁屏认证本地配置模型。
    * **关键字段**: `ukeyUnlockEnabled`，控制锁屏 UKey 认证入口是否启用。
  * `LockScreenUKeyDevice`
    * **所在文件**: `entry/src/main/ets/models/identity/lockscreen-auth/LockScreenAuthModels.ets`
    * **业务作用**: MDM 侧读取到的候选 USB/UKey 管理标识，不承载私钥、PIN 或认证结果。
    * **关键字段**: `fingerprint`、`deviceId`、`vendorId`、`productId`、`serial`、`deviceName`、`stableIdentifier`。
  * `LockScreenUKeyBinding`
    * **所在文件**: `entry/src/main/ets/models/identity/lockscreen-auth/LockScreenAuthModels.ets`
    * **业务作用**: 首把 UKey 注册成功后的本地绑定记录，用于阻止后续其它 USB/UKey 替换或重复注册。
    * **关键字段**: `fingerprint`、`deviceId`、`userId`、`credentialIdHex`、`pluginInfo`、`boundAt`。
  * `DEFAULT_LOCKSCREEN_AUTH_CONFIG`
    * **所在文件**: `entry/src/main/ets/models/identity/lockscreen-auth/LockScreenAuthModels.ets`
    * **业务作用**: 新安装、历史配置缺省或配置读取失败时的默认值。
    * **关键字段**: `ukeyUnlockEnabled = true`。
  * `IdentitySettingsState`
    * **所在文件**: `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`
    * **业务作用**: 身份鉴别页面编辑态和派生保存状态。
    * **关键字段**: `adminActivated`, `minLengthIndex`, `requireUppercase / requireLowercase / requireDigit / requireSpecialChar`, `validityPeriodIndex`, `customValidityPeriodText`, `hasChanges`, `canSave`。
  * `IdentitySettingsActionRequest`
    * **所在文件**: `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`

* **核心业务用例 (Use Cases)**:
  * `IdentityPasswordPolicyMapper.toConfig(policy)`: 将系统 `securityManager.PasswordPolicy` 转为页面可编辑的 `PasswordComplexityConfig`。
    * **副作用**: 无。
    * **失败策略**: `policy` 为空时返回 `null`；无法解析最小长度时使用 `0`；通过已知 lookahead 片段识别大小写、数字和特殊字符要求。
  * `IdentityPasswordPolicyMapper.toPasswordPolicy(config)`: 将页面配置转为系统 `PasswordPolicy`。
    * **副作用**: 无。
    * **失败策略**: `validityPeriod` 为数字时转换为毫秒；否则不写入 `validityPeriod` 字段。
  * `IdentityPasswordPolicyMapper.buildComplexityRegex(config)`: 构建系统口令复杂度正则。
    * **副作用**: 无。
    * **失败策略**: 未开启任何字符类型要求时使用 `^.{minLength,}$`；开启要求时追加对应 lookahead 并构建字符集。
  * `IdentityPasswordPolicyMapper.buildDescription(config)`: 构建系统可读的策略描述。
    * **副作用**: 无。
    * **失败策略**: 按最小长度、大写、小写、数字、特殊字符的稳定顺序拼接说明。
  * `IdentityService.getPasswordPolicyWithResult()`: 读取当前系统口令策略。
    * **副作用**: 调用 `securityManager.getPasswordPolicy(ADMIN_WANT)`。
    * **失败策略**: 捕获异常并通过 `OperationResultUtils.buildFailureResult(...)` 包装为 `OperationResult<PasswordPolicy | null>`，失败时 `data` 兜底为 `null`。
  * `IdentityService.setPasswordPolicyWithResult(config)`: 下发系统口令策略。
    * **副作用**: 调用 `securityManager.setPasswordPolicy(ADMIN_WANT, policy)`。
    * **失败策略**: 捕获异常并包装为 `OperationResult<void>`；管理员缺失由统一 failure result 携带 `isAdminMissing`。
  * `AuthService.checkAuthMethodAvailability(authMethod)`: 检查 PIN / 指纹当前是否可用。
    * **副作用**: 调用 `UserAuthAdapter.getAvailableStatus(...)`，间接访问 `userAuth.getAvailableStatus(authType, ATL1)`。
    * **失败策略**: 不支持的认证方式返回 `METHOD_UNSUPPORTED`；系统错误码 `12500010 / 12500006 / 12500013 / 201 / 401` 通过 `SystemErrorCodes` 中的命名常量映射为项目文案。
  * `AuthService.authenticate(authMethod, challenge?)`: 拉起系统认证并返回认证执行结果。
    * **副作用**: 无有效 challenge 时通过 `cryptoFramework.createRandom().generateRandomSync(8)` 生成随机 challenge；创建 `userAuth.UserAuthInstance`，注册 `result` 监听并调用 `start()` 拉起系统认证 UI。
    * **失败策略**: 不支持、challenge 生成失败或系统不可用返回 `reason='unavailable'`；认证失败、取消或非可用性错误返回 `reason='failed'`；成功或回调结束后尝试主动 `off('result')`。
  * `AuthService.getAuthMethodLabel(method)`: 获取认证方式展示名称。
    * **副作用**: 无。
    * **失败策略**: 未知枚举返回通用“认证方式”文本。
  * `UserAuthAdapter.getAvailableStatus(authType)`: 封装 `userAuth.getAvailableStatus(authType, ATL1)`。
    * **副作用**: 调用系统认证可用性 API。
    * **失败策略**: 捕获 `BusinessError`，返回结构化 `UserAuthStatusResult`。
  * `UserAuthAdapter.getUserAuthInstance(request)`: 封装 `userAuth.getUserAuthInstance(authParam, widgetParam)`。
    * **副作用**: 创建系统认证实例。
    * **失败策略**: 捕获 `BusinessError`，返回结构化 `UserAuthInstanceResult`。
  * `LockScreenUKeyDeviceService.getCandidateDevices()`: 通过 DDK 枚举当前 USB 设备并映射为 MDM 侧 UKey 管理候选。
    * **副作用**: 调用 `@kit.DriverDevelopmentKit.deviceManager.queryDevices(deviceManager.BusType.USB)`；随后调用 `@kit.BasicServicesKit.usbManager.getDevices()` 补充 serial、class、manufacturerName、productName。
    * **失败策略**: DDK 枚举失败返回失败结果；过滤 USB Hub；优先使用 USB_STORAGE 候选，若无存储类候选则使用其它非 Hub USB 候选；优先用 `vendorId/productId/serial` 生成稳定管理标识，缺失 serial 时使用 `vendorId/productId/DDK description/deviceName/DDK deviceId` 生成弱标识并标记 `stableIdentifier=false`。
  * `LockScreenUKeyBindingRepository.loadBinding(context)` / `saveBinding(context, binding)`: 读写首把 UKey 绑定记录。
    * **副作用**: 读写 Preferences store `lockscreen_auth_settings`。
    * **失败策略**: 读取失败或 JSON 无效时返回 `null`；保存失败返回 `false`。
  * `LockScreenCustomAuthEnrollmentService.onUKeyAttached(context)` / `ensureFirstUKeyRegistered(context)`: 确保当前锁屏 UKey 认证存在首把绑定和活动凭据。
    * **副作用**: 读取本地绑定和活动凭据；必要时枚举 USB 设备，按测试 HAP 当前默认目标账号 `100` 调用 `@mdm.openSession/addUserCustomCredential/closeSession` 发起 CustomAuth 凭据注册，并保存绑定与活动凭据记录。
    * **失败策略**: 已有绑定时不再注册第二把；同一把首把重插且活动凭据缺失时补注册；无候选或多候选时失败且不调用注册接口；注册失败或未返回 `credentialId` 时不保存活动凭据；关闭开关不调用该流程。
  * `LockScreenCustomAuthEnrollmentService.onUKeyDetached(context)`: 首把 UKey 不在场时删除本应用管理的活动凭据。
    * **副作用**: 读取本地绑定和活动凭据；枚举当前 USB 设备；首把不在场且存在活动凭据时调用 `@mdm.openSession/deleteUserCustomCredential/closeSession`，删除成功后清空活动凭据记录。
    * **失败策略**: 无可信绑定、活动凭据缺失或首把仍在场时不删除；删除失败保留活动凭据以便后续重试；不删除可信绑定。
  * `LockScreenUKeyRuntimeEnrollmentConsumer.consume(event)`: 响应外设运行时 USB attach / detach 事件。
    * **副作用**: 读取锁屏 UKey 开关；开关开启时按事件类型分别调用 `LockScreenCustomAuthEnrollmentService.onUKeyAttached(context)` 或 `onUKeyDetached(context)`。
    * **失败策略**: 开关关闭、context 缺失或已有同类处理进行中时跳过；注册/删除失败仅记录日志，不弹窗；始终返回 `null`，不写外设 trace。
  * `PreferencesLockScreenAuthRepository.loadConfig(context)`: 读取锁屏认证配置。
    * **副作用**: 读取 Preferences store `lockscreen_auth_settings`。
    * **失败策略**: 读取失败或历史配置没有 `lockscreen_ukey_auth_enabled` 时返回默认开启。
  * `PreferencesLockScreenAuthRepository.saveConfig(context, config)`: 保存锁屏认证配置。
    * **副作用**: 写入 Preferences key `lockscreen_ukey_auth_enabled`。
    * **失败策略**: 写入失败或异常时返回 `false`，ViewModel 回退开关状态并提示保存失败。
* **内部架构设计**:
  * **Repository**: 口令策略不维护本地持久化仓库，真源是系统 MDM `PasswordPolicy`；锁屏认证配置由 `PreferencesLockScreenAuthRepository` 使用独立 Preferences store 维护。
  * **Mapper/Ports**: `IdentityPasswordPolicyMapper` 承担 `PasswordComplexityConfig` 与 `PasswordPolicy` 的 mapper；`UserAuthAdapter` 是系统认证 API port。

`IdentityService`、`IdentityPasswordPolicyMapper`、`AuthService`、`UserAuthAdapter` 的职责边界必须保持清晰：

- `IdentityService` 管“口令策略读写”：只封装 `securityManager.getPasswordPolicy` / `setPasswordPolicy` 调用、taskpool 兜底和异常结果包装。
- `IdentityPasswordPolicyMapper` 管“口令策略如何转换”：最少几位、是否必须包含字符类型、密码多久过期，以及这些配置如何转换为 MDM `PasswordPolicy`。
- `AuthService` 管“是否能证明当前操作者本人”：PIN / 指纹是否可用、如何发起认证、系统认证错误如何映射为业务结果。
- `UserAuthAdapter` 管“如何调用系统认证 API”：只做 `UserAuthenticationKit` 调用与异常结构化，不承载业务文案和业务判断。
- `LockScreenAuthRepository` 管“锁屏认证本地配置如何持久化”：只读写 UKey 锁屏认证开关，不调用真实 UKey、UserAuth、CustomAuth 或 SystemUI。
- `LockScreenUKeyBindingRepository` 管“首把 UKey 可信绑定如何持久化”：只保存 MDM 管理标识与历史 credentialId，不保存私钥、PIN 或认证结果；拔出时不删除。
- `LockScreenUKeyActiveCredentialRepository` 管“当前活动 CustomAuth 凭据如何持久化”：只保存本应用最近注入的活动凭据 ID 和对应 UKey 标识，用于拔出删除，不替代可信绑定。
- `LockScreenUKeyDeviceService` 管“当前 USB/UKey 管理候选如何读取”：默认通过 DDK 发现 USB 设备，只做枚举和管理标识构造，不做认证；`usbManager` 仅用于补充详情。
- `LockScreenCustomAuthEnrollmentService` 管“何时注册或删除首把 UKey 活动凭据”：已有绑定不注册第二把；无绑定且当前恰好一把候选时才调用系统凭据注册；首把拔出时删除活动凭据但保留绑定。
- `LockScreenUKeyRuntimeEnrollmentConsumer` 管“插拔事件是否触发凭据生命周期”：只消费 USB attach / detach 事件并委托注册或删除服务，不写外设连接记录、不改变外设策略判定。
- `LockScreenAuthViewModel` 管“锁屏认证设置区页面状态”：初始化、即时保存、后台注册和失败回退。

`AuthService` 当前映射的认证错误码如下，具体数字统一从 `entry/src/main/ets/constants/SystemErrorCodes.ets` 读取：

| 错误码 | 语义 | 处理结果 |
|---|---|---|
| `12500010` | 未录入或凭据不满足要求 | 返回未录入提示 |
| `12500006` | 认证等级不足 | 返回认证等级不足提示 |
| `12500013` | PIN 过期 | 返回 PIN 过期提示 |
| `201` | 权限不足 | 返回认证权限不足提示 |
| `401` | 参数错误 | 返回认证参数错误提示 |
| 其它 | 未知系统错误 | 优先使用系统 fallback message，否则返回统一不可用提示 |

#### 4.3 ViewModel (视图模型层)
* **真实文件路径**:
  * `entry/src/main/ets/viewmodels/identity/settings/IdentitySettingsViewModel.ets`
  * `entry/src/main/ets/viewmodels/identity/lockscreen-auth/LockScreenAuthViewModel.ets`
* **状态分发逻辑**:
  * `initialize()`: 设置 loading，刷新管理员状态；管理员未激活时重置为默认策略，管理员已激活时通过 `IdentityService.getPasswordPolicyWithResult()` 读取系统策略并回填页面。该方法由页面在共享 ViewModel 未初始化时调用，避免路由返回时覆盖当前编辑态。
  * `reset()`: 调用 `applyPolicy(null)` 回到默认页面编辑态。
  * `setMinLengthIndex(index)`: 当 index 落在 `PASSWORD_LENGTH_OPTIONS` 范围内时更新最小长度索引，并刷新 `hasChanges / canSave`。
  * `setRequirement(key, value)`: 更新大写、小写、数字或特殊字符开关，并刷新派生状态。
  * `setValidityPeriodIndex(index)`: 更新有效期选项；切离自定义时清空 `customValidityPeriodText`。
  * `setCustomValidityPeriodText(value)`: 自定义有效期只保留数字字符。
  * `requestSave(skipImmediateExpiryConfirm=false)`: 保存入口。先重新探测管理员状态，再构建配置；自定义有效期非法时返回失败；自定义 `0` 天表示马上过期，未跳过确认时返回 `confirm`；否则进入 `executeSave(config)`。
  * `executeSave(config)`: 重新读取当前系统策略，与目标配置比较；无变化返回失败提示；有变化则调用 `IdentityService.setPasswordPolicyWithResult(config)`，成功后更新 `initialState` 并清除 dirty 状态。
  * `applyConfig(state, config, policy?)`: 将 `PasswordComplexityConfig` 和原始 `PasswordPolicy.validityPeriod` 映射为页面状态。系统未配置策略时 `IdentityPasswordPolicyMapper.toConfig(null)` 返回 `null`，页面使用默认“永久”；系统返回空策略且携带 `validityPeriod = 0` 时也按未配置处理，避免把设备默认状态误显示为马上过期；半年、一年、永久通过 `IdentityModels.ets` 中的毫秒天数候选集合识别；非空系统策略或保存目标中存在 `validityPeriod = 0` 时，回显为自定义 `0`，保持马上过期语义；其它正数回填自定义天数。
  * `LockScreenAuthViewModel.initialize(context)`: 读取锁屏认证配置和首把 UKey 绑定并回显，页面只在共享 ViewModel 未初始化时调用；配置为开启且没有绑定时后台尝试注册。
  * `LockScreenAuthViewModel.setUKeyUnlockEnabled(context, enabled)`: 即时保存锁屏 UKey 解锁开关；成功后更新状态，失败时回退原值并返回结果动作；切换为开启时尝试首把 UKey 注册。

ViewModel 内部有两个状态基线：`initialState` 是最近一次初始化或保存成功后的快照，`state` 是当前编辑态。`hasChanges` 只比较这两者，不直接依赖页面控件。

口令策略回显链路如下：

```text
IdentityPage.aboutToAppear()
  -> IdentityPage.initializeViewModel()
  -> 若共享 ViewModel 未初始化，则调用 IdentitySettingsViewModel.initialize()
  -> EnterpriseAdminService.probeActivation()
  -> IdentityService.getPasswordPolicyWithResult()
  -> securityManager.getPasswordPolicy(ADMIN_WANT)
  -> IdentityPasswordPolicyMapper.toConfig(policy)
  -> IdentitySettingsViewModel.applyPolicy(policy)
  -> 页面控件读取 viewModel.state 回显
```

回显异常策略：

- 企业管理员未激活：不读取系统策略，回退默认状态。
- 读取策略失败：回退默认状态。
- 系统返回空策略：`IdentityPasswordPolicyMapper.toConfig(null)` 返回 `null`，ViewModel 使用默认状态并回显默认“永久”；若系统返回空 `PasswordPolicy` 但默认填充 `validityPeriod = 0`，也视为未配置而不是马上过期。默认“永久”由永久预设下发和识别，不使用 `0` 表示。
- “永久”不是 `0` 天，而是当前实现按约 100 年有效期下发，回显时通过毫秒天数候选集合识别；`0` 天按产品规格表示马上过期。

  - `ohos.permission.ENTERPRISE_MANAGE_SECURITY`
  - `ohos.permission.ENTERPRISE_SET_ACCOUNT_POLICY`
  - `ohos.permission.ENTERPRISE_ADMIN_MANAGE`
  - `ohos.permission.ACCESS_BIOMETRIC`
  - `ohos.permission.MANAGE_LOCAL_ACCOUNTS`

  - `ohos.permission.USE_USER_IDM`

  - 锁屏 UKey 开关本身不需要 `ACCESS_USER_AUTH_INTERNAL`；但开启后的首把 UKey 凭据注册需要系统身份凭据管理权限，权限需同步 `module.json5` 与签名 profile。
- **异常兜底策略**:
  - 企业管理员未激活时，`initialize()` 不读取系统口令策略，页面回退默认编辑态；保存时返回失败结果，提示先激活企业管理员。
  - 读取口令策略失败时，ViewModel 记录告警并回退默认编辑态，不阻塞页面显示。
  - 系统未配置口令策略时回显为默认“永久”；默认“永久”不是 `0`；系统空策略携带 `validityPeriod = 0` 仍按未配置处理；非空系统策略或保存目标中的 `validityPeriod = 0` 回显为自定义 `0` 天，表示马上过期。
  - 自定义有效期为空或非数字时，`requestSave()` 返回失败提示；页面不调用系统保存。
  - 自定义有效期为 `0` 时表示马上过期，首次保存返回 `confirm`，用户确认后才继续保存；回读到非空系统策略的 `0` 天时也回显为自定义 `0`，不得映射成永久有效；回读到空系统策略的 `0` 天时视为未配置并回显默认“永久”。
  - 保存前重新读取当前系统策略并与目标配置比较；无变化时返回“无变更”，不调用 `setPasswordPolicy`。
  - `setPasswordPolicy` 失败时，管理员缺失失败优先显示管理员未激活类提示，其它失败显示重试提示。
  - PIN / 指纹认证不可用时，`AuthService` 将系统错误码映射为业务文案；认证失败或取消返回 `reason='failed'`，系统能力缺失返回 `reason='unavailable'`。
  - 随机 challenge 生成失败时，`AuthService.authenticate()` 返回认证能力不可用，不继续创建认证实例。
  - 锁屏 UKey 认证开关读取失败时回退默认开启；保存失败时 ViewModel 回退到切换前状态并提示保存失败。
  - 关闭锁屏 UKey 认证开关不删除系统或认证器中的 UKey 凭据，也不直接调用系统锁屏能力。
  - SecurityTool 只通过 DDK `deviceManager.queryDevices(BusType.USB)` 和 `usbManager.getDevices()` 读取当前 USB 设备描述符并生成 MDM 管理标识；不读取私钥、PIN、证书私有材料，不打开 USB pipe 做认证交互。
  - UKey 管理标识优先使用 `VID/PID + serial`；设备不暴露 `serial` 时，兜底使用 `VID/PID + DDK description + deviceName + DDK deviceId`，其中 `deviceName` 仅作为辅助区分信息。
  - 无候选 USB/UKey 或同时存在多把候选时，不发起 CustomAuth 凭据注册，避免误绑定。
  - 已有首把 UKey 绑定时，不再为后续插入的其它 USB/UKey 注册新凭据，也不覆盖本地绑定。
  - USB attach 事件触发的注册、USB detach 事件触发的活动凭据删除只记录日志，不弹窗，不写外设 trace；与页面初始化或开关开启并发时由 `LockScreenCustomAuthEnrollmentService` 的全局 pending 保护合并为一次处理。
  - CustomAuth 凭据注册失败、未返回 `credentialId` 或本地绑定保存失败时，ViewModel 保持开关状态但记录失败 message；手动开启场景通过结果弹窗提示失败。
  - CustomAuthenticator 当前参考实现的 `UKeyProvider` 使用 `AppStorage` 模拟在场 UKey、绑定关系和 PIN；生产接入时应替换为硬件 backend，由 DDK/USB/证书能力完成枚举、唯一标识、PIN/证书校验、模板绑定和换新策略，并关闭非明确换新流程下的“顺序尝试/自动轮换”回退，确保第二把 UKey 不能解锁。

### 5.1 实施步骤与测试验收 (Implementation & Acceptance)

- **实施步骤**:
  - 涉及口令策略、复杂度映射、认证能力、企业管理员激活判断、页面交互或权限变化时，先更新本文档的目标范围、状态模型、职责边界、异常兜底和验收口径。
  - 口令配置模型、文案和映射规则先同步 `IdentityModels.ets`、`IdentityStrings.ets`、`IdentityPasswordPolicyMapper.ets`，再调整系统读写服务。
  - 系统口令策略读写统一经 `IdentityService.ets`，认证能力统一经 `AuthService.ets`；页面和 ViewModel 不直接调用 MDM 或 userAuth API。
  - 锁屏认证开关读写统一经 `LockScreenAuthRepository.ets`；首把 UKey 绑定统一经 `LockScreenUKeyBindingRepository.ets`；CustomAuth 注册统一经 `LockScreenCustomAuthEnrollmentService.ets`；页面不直接访问 Preferences、USB、系统 UserAuth API 或 CustomAuthenticator IPC。
  - USB 插入触发注册、USB 拔出触发活动凭据删除统一经 `LockScreenUKeyRuntimeEnrollmentConsumer.ets` 接入外设运行时事件；该 Consumer 不得写外设 trace 或下发外设黑白名单策略。
  - 自定义有效期、无变更保存、管理员未激活和认证不可用等用户可见流程变化，必须同步页面提示、ViewModel 结果类型和测试用例。
  - 新增或调整企业管理/生物认证权限时，同步 `entry/src/main/module.json5`、`hapsigner/UnsgnedDebugProfileTemplate.json` 并按签名流程重新生成 p7b。
  - 新增或调整锁屏 UKey 凭据注册权限时，同步 `entry/src/main/module.json5`、`hapsigner/UnsgnedDebugProfileTemplate.json` 并按签名流程重新生成 p7b。
  - 新增或调整 DDK 设备发现或 native DDK 权限时，同步 `entry/src/main/module.json5`、`hapsigner/UnsgnedDebugProfileTemplate.json`、`AGENTS.md` 并按签名流程重新生成 p7b。
- **测试覆盖**:
  - UT: `entry/src/test/identity/*.test.ets` 覆盖口令策略映射、Service、ViewModel 和页面状态；`entry/src/test/auth/*.test.ets` 覆盖 PIN / 指纹认证能力映射。
  - UT: `entry/src/test/identity/lockscreen-auth.test.ets` 覆盖锁屏 UKey 开关默认开启、保存关闭、保存失败回退和 ViewModel 共享状态。
  - UT: `entry/src/test/identity/lockscreen-auth.test.ets` 覆盖首把 UKey 注册：无绑定且一把候选时调用注册并保存绑定；已有绑定时不注册第二把；无候选或多候选时不调用注册。
  - UT: `entry/src/test/identity/lockscreen-auth.test.ets` 覆盖 USB attach / detach 事件：开关开启时分别触发注册和删除服务，开关关闭时跳过。
  - ohosTest: `entry/src/ohosTest/ets/test/simple/RouteAction.test.ets` 通过 `route_action` 场景覆盖身份鉴别入口和页面可达性。
  - E2E: `scripts/e2e/cases/identity/validity_section.json`、`password_policy.json`、`complexity_requirements.json`、`domain_policy.json` 覆盖有效期、口令策略、复杂度和域策略展示。
- **验收口径**:
  - 企业管理员未激活时页面可显示默认编辑态，但保存必须失败并提示激活管理员。
  - 自定义有效期为空或非数字时不调用系统保存；自定义有效期为 `0` 时必须先二次确认，非空策略回显也必须保留为自定义 `0`；空策略携带 `0` 时按默认“永久”回显。
  - 保存前应比较当前系统策略和目标配置，无变化不调用 `setPasswordPolicy`；认证能力不可用时返回明确业务文案。
  - 锁屏 UKey 认证开关默认开启；关闭后关机重启或重新进入页面仍保持关闭；历史无配置时按开启回显。
  - 切换锁屏 UKey 认证开关不触发口令策略保存按钮状态变化，也不影响应用启动认证。
  - 页面只保留一个 `UKey 锁屏认证` 开关；不得再暴露验证按钮、验证结果、绑定、切换模拟 UKey、删除 fake 凭据等调试控件。
  - 关闭锁屏 UKey 认证开关后，SecurityTool 不删除系统或认证器中已有 UKey 凭据。
  - 无首把绑定且当前恰好插入一把候选 USB/UKey 时，开启锁屏 UKey 认证会尝试注册 CustomAuth 凭据；成功后保存 `credentialIdHex` 与该 UKey 管理标识。
  - 应用运行时收到 USB attach 事件且开关开启时，应按同一首把规则尝试注册；关闭开关或已有绑定时不应为后续 UKey 注册新凭据。
  - 应用运行时收到 USB detach 事件且开关开启时，若首把 UKey 已不在场且存在活动凭据，应调用删除凭据并清空活动凭据记录；可信绑定必须保留。
  - 已存在首把 UKey 绑定后，后续插入其它 USB/UKey 不应触发新凭据注册，也不应覆盖本地绑定。
  - 当前插入 0 把或多把候选 USB/UKey 时，不调用 CustomAuth 注册接口。
