- **维护规则**: 本文档为帮助与反馈页面的设计契约。修改页面入口、信息分区、展开/收起交互、联系信息、帮助文案、测试覆盖口径时，必须同步更新本文档。

## 1. 业务概述与对外接口 (Overview & Public Interfaces)
- **核心目标**: 为用户提供产品使用指南、常见问题和反馈联系方式，支撑演示、验收和日常排障。
- **入口路由 (Route Entry)**: `RouteIds.HELP_FEEDBACK = 'help-feedback'`。由顶部菜单和安全总览快捷入口进入，主页面渲染 `HelpFeedbackPage`。
- **对外暴露能力 (Public APIs)**: 无 Service / Repository 对外接口；页面仅消费静态文案常量和公共 UI 组件。
- **业务边界**:
  - ✅ **包含**: 使用指南展示、FAQ 展示、单项展开/收起、反馈邮箱展示、返回安全总览。
  - ❌ **不包含**: 在线客服、问题提交表单、附件上传、远程日志上报、服务端工单系统、动态内容拉取。

## 2. 状态与数据流 (Data Flow & State)
> **设计原则**: 页面仅维护 UI 展开态，不引入 ViewModel、Service 或持久化。

- **核心业务状态 (Core Business State)**:
  - 无核心业务状态。
- **页面 UI 状态**:
  - `expandedGuide: number`: 当前展开的使用指南项索引，初始为 `-1`。
  - `expandedFaq: number`: 当前展开的 FAQ 项索引，初始为 `-1`。
- **关键流转路径**:
  - 顶部菜单帮助入口 / 首页快捷入口 -> `MainPage.setCurrentPage(RouteIds.HELP_FEEDBACK)` -> `HelpFeedbackPage` 渲染。
  - 点击使用指南项 -> 切换 `expandedGuide` -> 同一组内只展开一个指南项，再次点击收起。
  - 点击 FAQ 项 -> 切换 `expandedFaq` -> 同一组内只展开一个 FAQ 项，再次点击收起。
  - 点击返回 -> `onBack()` -> `MainPage` 切回 `RouteIds.DASHBOARD`。

## 3. 模块结构与组件设计 (Module Components)

### 【核心层】(Core MVVM & Domain Layers)

#### 3.1 Model & Types (核心数据模型与类型)
* **核心实体与依赖路径**:
  * `GuideItemString`
    * **所在文件**: `entry/src/main/ets/constants/modules/HelpFeedbackStrings.ets`
    * **业务作用**: 定义使用指南标题和描述。
    * **关键字段**: `title`、`description`。
  * `FaqItemString`
    * **所在文件**: `entry/src/main/ets/constants/modules/HelpFeedbackStrings.ets`
    * **业务作用**: 定义 FAQ 问题和答案。
    * **关键字段**: `question`、`answer`。
  * `ExpandableInfoItem`
    * **所在文件**: `entry/src/main/ets/views/help-feedback/overview/HelpFeedbackPage.ets`
    * **业务作用**: 页面内把指南和 FAQ 统一映射成可展开列表项。
    * **关键字段**: `title`、`content`。

#### 3.2 Service / Domain (领域业务层)
* **真实文件路径**: 无。
* **核心业务用例 (Use Cases)**:
  * 无业务服务调用。页面内容来自静态常量，交互只更新页面局部展开态。
* **内部架构设计**:
  * **Repository**: 无。
  * **Mapper/Ports**: 页面内将 `GuideItemString` / `FaqItemString` 映射为 `ExpandableInfoItem`。

#### 3.3 ViewModel (视图模型层)
* **真实文件路径**: 无。
* **状态分发逻辑**:
  * 当前页面没有独立 ViewModel；`expandedGuide` 和 `expandedFaq` 是页面 UI 局部态，不作为业务状态真源。

#### 3.4 View / Page (页面视图层)
* **真实文件路径**: `entry/src/main/ets/views/help-feedback/overview/HelpFeedbackPage.ets`
* **结构说明**:
  * `HelpFeedbackPage`: 帮助与反馈页面入口。使用 `SubPageHeader` 展示标题和返回操作，主体使用三个 `SectionCard` 展示“使用指南”“常见问题”“联系与反馈”。
  * `guideSectionContent()`: 将 `HELP_FEEDBACK_GUIDE_ITEMS` 映射成可展开指南列表。
  * `faqSectionContent()`: 将 `HELP_FEEDBACK_FAQ_ITEMS` 映射成可展开 FAQ 列表。
  * `contactSectionContent()`: 展示反馈描述和反馈邮箱。

#### 3.5 Components (可复用组件层)
* **真实文件路径**: `entry/src/main/ets/views/help-feedback/overview/HelpFeedbackPage.ets`
* **结构说明**:
  * `ExpandableInfoList`: 页面内私有组件。**入参/事件**: `items`、`expandedIndex`、`currentTheme`、`onToggle(index)`。
* **复用公共组件**:
  * `entry/src/main/ets/components/SubPageHeader.ets`: 页面标题、说明和返回入口。
  * `entry/src/main/ets/components/SettingsSectionCard.ets`: 分区卡片容器。

### 【基础设施与扩展层】(Infrastructure & Extensions)

#### 3.6 Storage / Database (持久化)
* **涉及修改**: 无。帮助内容不落库、不缓存、不持久化展开态。

#### 3.7 Contracts / IPC (通信契约)
* **真实文件路径**: 无。

#### 3.8 Constants & Utils (业务常量与工具)
* **说明**:
  * `entry/src/main/ets/constants/modules/HelpFeedbackStrings.ets`: 页面标题、副标题、分区标题、联系信息、展开/收起文案、使用指南和 FAQ 静态内容。
  * `entry/src/main/ets/constants/RouteIds.ets`: `HELP_FEEDBACK` 路由 ID。
  * `entry/src/main/ets/constants/StringCatalog.ets`: `ModuleText.HELP_FEEDBACK` 页面标题。

#### 3.9 Ability / Runtime (系统入口)
* **说明**:
  * `entry/src/main/ets/pages/MainPage.ets`: 持有 `help-feedback` 路由渲染逻辑；顶部菜单和安全总览快捷入口均最终通过主页面切换到本页面。
  * 页面不依赖 UIAbility 生命周期、企业管理员激活、系统权限或后台运行时服务。

## 4. 异常处理与系统依赖 (Dependencies & Errors)
- **关键系统 API**: 无。
- **系统权限**: 无新增权限。
- **异常兜底策略**:
  - 静态数组为空时，对应分区仅显示空列表；当前代码不额外展示错误态。
  - 联系方式为静态邮箱，不触发邮件客户端或外部跳转，因此没有外部应用失败链路。
  - 展开索引异常时由点击逻辑覆盖为当前索引或 `-1`，不影响其它模块状态。

### 4.1 实施步骤与测试验收 (Implementation & Acceptance)

- **实施步骤**:
  - 修改帮助指南、FAQ、联系信息、入口或展开交互前，先更新本文档中的目标范围、页面结构、状态模型和异常兜底口径。
  - 文案和静态数据统一维护在 `HelpFeedbackStrings.ets`；页面 `HelpFeedbackPage.ets` 只负责渲染、折叠展开和返回/导航回调。
  - 路由或入口变化需要同步 `RouteIds.ets`、`StringCatalog.ets`、`MainPage.ets`、安全总览快捷入口和导航 E2E 用例。
  - 本模块当前不引入 ViewModel、Service、Repository、Storage 或系统权限；若后续接入真实反馈提交能力，必须先扩展本文档的职责、数据和权限章节。
- **测试覆盖**:
  - UT: `entry/src/test/help/strings.test.ets` 覆盖静态分区、指南和 FAQ 文案完整性。
  - ohosTest: `entry/src/ohosTest/ets/test/simple/RouteAction.test.ets` 通过 `route_action` 场景覆盖帮助与反馈入口；`entry/src/ohosTest/ets/test/theme/theme-menu-popup.test.ets` 覆盖顶部菜单入口。
  - E2E: `scripts/e2e/cases/navigation/help_feedback.json` 与 `help_feedback_details.json` 覆盖导航进入、指南/FAQ 详情和返回链路。
- **验收口径**:
  - 顶部菜单和安全总览快捷入口均能进入帮助与反馈页，页面返回不破坏主页面当前导航状态。
  - 使用指南、FAQ 和联系方式内容完整可见，展开/收起只影响当前条目，不影响其它模块状态。
  - 页面不触发外部应用、不写本地存储、不申请权限，离线和未激活企业管理员状态下仍可展示。
