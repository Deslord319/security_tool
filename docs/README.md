# 文档目录说明

当前 `docs/` 目录采用中文精简结构，按“设计、测试、发布、排障、模板、专项资料”组织。

## 目录结构

- `01-总体设计/`：系统级设计资料，如产品设计图、全局审计报告。
- `02-模块设计/`：面向具体模块的设计与专项分析资料。
- `03-测试文档/`：测试设计、测试指南、测试报告、阶段性测试资料。
- `04-构建发布/`：构建、安装、提交流程和界面问题处理流程说明。
- `05-问题排查/`：故障记录、诊断截图、评审材料和排查辅助资料。
- `06-文档模板/`：统一的文档模板。
- `07-好代码资料/`：好代码说明、评选文档和配套素材。

## 常用入口

- `[构建与安装指南](./04-构建发布/构建与安装指南.md)`
- `[测试检查清单](./03-测试文档/测试检查清单.md)`

## 维护约定

- 新增文档优先使用中文文件名。
- 与模块强相关的资料优先归入 `02-模块设计/` 或 `03-测试文档/`。
- “好代码”相关材料统一放在 `07-好代码资料/`，不要混放到其他目录。
