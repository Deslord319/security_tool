# 防火墙 IP 规则规格唯一真相

> 状态：Frozen  
> 日期：2026-04-20  
> 适用范围：仅用于“防火墙管理 > 自定义模式 > 新增/编辑 IP 规则”的规格冻结  
> 唯一判定依据：本文档

## 0. 背景+目的

当前防火墙“自定义模式”中的 IP 规则录入与展示存在三个核心问题：

1. IP 规则的主字段没有严格按方向区分，入站和出站的语义容易混淆
2. IP 地址与端口输入能力不足，无法覆盖当前测试中需要的 IP 段和端口混写场景
3. `ALL` 的展示、回显、保存和冲突判断语义没有统一

本次规格冻结的目标是：

1. 将 IP 规则的方向语义固定下来
2. 将 IP 输入、端口输入和 `ALL` 语义固定下来
3. 将冲突检测与展示回显口径固定下来
4. 为后续实施提供唯一真相，避免实现过程中继续漂移

本次只讨论 IP 规则，不调整 `DOMAIN` 和 `DNS` 规则主语义。

## 1. 规格结论

### 1.1 字段语义

IP 规则按方向展示不同字段：

- 入站：`本地地址 + 本地端口`
- 出站：`远端地址 + 远端端口`

当前规格不开放另一侧字段的录入入口，不做“高级字段”预留设计。

### 1.2 IP 输入规格

IP 地址使用单输入框，业务层负责解析。

支持以下三种合法输入：

- 单 IP
- CIDR
- `startIp-endIp`

输入规则：

- 允许首尾空格，保存前自动 `trim`
- 若字符串包含 `-`，按 IP 段解析
- 否则按单 IP / CIDR 解析
- 单 IP 不自动回显成 `/32`
- CIDR 按原语义回显
- IP 段按 `startIp-endIp` 回显

合法性要求：

- 单 IP 必须是合法 IPv4
- CIDR 的 mask 范围必须是 `0-32`
- IP 段必须满足 `startIp <= endIp`
- 不接受残缺格式或多短横线格式

示例：

- `110.110.110.110`
- `8.8.8.0/24`
- `110.110.110.1-110.110.110.254`

非法示例：

- `110.110.110.1-`
- `-110.110.110.254`
- `8.8.8.0/33`

### 1.3 端口输入规格

端口使用单输入框。

支持以下合法输入：

- 单端口
- 端口范围
- 单端口 + 范围混写

输入规则：

- 使用逗号分隔
- 允许首尾空格
- 允许逗号后的空格
- `"ALL"` 字符串本身不允许输入
- 空值表示 `ALL`

归一化规则：

- 去重
- 升序排序
- 仅做完全重复项去重，不做区间合并
- `80,443,1000-2000` 计为 `3` 项
- 上限以官方文档为准
- 按归一化后的有效项数计算

示例：

- `80`
- `80,443`
- `80,443,1000-2000`

非法示例：

- `ALL`
- `80,,443`
- `80,abc`
- `2000-1000`

### 1.4 ALL 语义

`ALL` 是展示和录入快捷语义，不是长期持久化语义。

规则如下：

- 地址为空表示 `ALL`
- 端口为空表示 `ALL`
- 底层字段不传
- 页面展示统一为 `ALL`
- 地址和端口同时为空时允许直接保存，不弹确认提示

### 1.5 展示回显

统一回显规则如下：

- 单 IP 原样回显
- CIDR 原样回显
- IP 段回显为 `startIp-endIp`
- 端口回显为归一化后的逗号字符串
- 空值统一回显为 `ALL`

适用位置：

- 规则列表
- 规则详情
- 编辑态回显
- 冲突提示中的旧规则展示

### 1.6 冲突检测口径

IP 规则冲突检测按方向使用不同主字段语义。

入站规则比较维度：

- 方向
- 协议
- 本地地址条件
- 本地端口条件

出站规则比较维度：

- 方向
- 协议
- 远端地址条件
- 远端端口条件

判定规则：

- 完全同键且动作相同 -> 重复规则
- 完全同键且动作相反 -> 硬冲突
- 范围有交集且动作相反 -> 重叠冲突

冲突判断需要覆盖以下交叉场景：

- 单 IP vs CIDR
- 单 IP vs IP 段
- CIDR vs IP 段
- IP 段 vs IP 段
- 单端口 vs 端口范围
- 端口范围 vs 端口范围
- 多端口集合之间的交集判断

## 2. 规格边界

本次冻结只覆盖以下内容：

1. IP 规则方向语义
2. IP 输入格式
3. 端口输入格式
4. `ALL` 语义
5. 展示回显
6. 冲突检测口径

本次不覆盖：

1. `DOMAIN` / `DNS` 规则重构
2. 持久化结构迁移
3. 规则下发主流程重构
4. 新的高级字段预留设计

## 3. 验收信号

完成后应满足：

1. 入站 IP 规则按本地地址 + 本地端口理解
2. 出站 IP 规则按远端地址 + 远端端口理解
3. IP 输入支持单 IP、CIDR 和 IP 段
4. 端口输入支持单端口、范围和混写
5. 空值统一显示为 `ALL`
6. 冲突检测口径与新字段语义一致
7. 规则列表、详情和编辑态回显一致
