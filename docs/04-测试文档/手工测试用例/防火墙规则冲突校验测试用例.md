# 防火墙规则冲突校验测试用例

## 1. 说明

本文档用于归档当前“防火墙规则 > 添加规则”链路的测试用例，覆盖以下三类判断：

- 重复规则：同键且动作相同，页面直接拦截
- 硬冲突：同键且动作相反，页面提示覆盖旧规则
- 重叠冲突：命中范围有交集且动作相反，页面提示风险后允许继续保存

当前实现说明：

- 规则来源为系统 `netFirewall`，不是应用本地缓存
- `DOMAIN` 规则只支持两类匹配：
  - 精确域名，例如 `www.baidu.com`
  - 单层子域通配，例如 `*.baidu.com`
- `*.baidu.com` 仅匹配单层子域：
  - 匹配 `www.baidu.com`
  - 不匹配 `baidu.com`
  - 不匹配 `a.b.baidu.com`
- `IP` 规则中的单个 IP 按 `/32` 处理

## 2. 页面真实提示文案

- 重复规则：`规则已存在，无需重复添加`
- 硬冲突弹窗标题：`存在冲突规则`
- 硬冲突弹窗内容：`检测到同一目标和方向下已存在相反动作的规则。覆盖后将删除旧规则并保存新规则。`
- 硬冲突主按钮：`覆盖`
- 重叠冲突弹窗标题：`存在重叠规则`
- 重叠冲突弹窗内容：`新规则与现有规则的命中范围存在重叠，实际生效结果可能受系统规则优先级影响。是否继续保存？`
- 重叠冲突主按钮：`继续保存`
- 直接新增成功：`规则添加成功`
- 覆盖成功：`规则已覆盖并生效`
- 参数错误：`规则参数不合法，请检查输入`
- 权限错误：`规则添加失败，请检查权限和企业管理员状态`

## 3. 测试前置条件

- 应用已安装到设备并成功激活企业管理员
- 进入“防火墙规则”页面
- 每条用例执行前，先清理与该用例相关的历史规则，避免互相干扰
- 若页面顶部出现历史冲突告警，先记录告警内容，再按用例步骤继续验证

## 4. 测试用例

### FW-CONFLICT-001 基础新增成功

- 前置条件：系统中不存在目标为 `www.baidu.com`、方向为出站的同类规则
- 操作步骤：
  1. 添加一条 `DOMAIN` 规则
  2. 域名填写 `www.baidu.com`
  3. 方向选择“出站”
  4. 动作选择“允许”
  5. 点击保存
- 预期提示：
  - toast：`规则添加成功`
- 预期结果：
  - 列表新增该规则
  - 页面不弹出冲突相关弹窗

### FW-CONFLICT-002 DOMAIN 重复规则

- 前置条件：已存在一条 `DOMAIN` 规则：`*.baidu.com`、出站、允许
- 操作步骤：
  1. 再添加一条 `DOMAIN` 规则
  2. 域名填写 `*.baidu.com`
  3. 方向选择“出站”
  4. 动作选择“允许”
  5. 点击保存
- 预期提示：
  - toast：`规则已存在，无需重复添加`
- 预期结果：
  - 不新增第二条规则
  - 列表中原规则保持不变

### FW-CONFLICT-003 DOMAIN 硬冲突覆盖

- 预期结果：

  - 新“拒绝”规则保存成功

## 5. 回归检查项

- 重复规则场景不会误弹“覆盖”或“继续保存”
- 硬冲突场景不会直接保存第二条规则
- 重叠冲突场景不会误删旧规则
- `*.baidu.com -> www.baidu.com` 必须识别为重叠冲突
- `*.baidu.com -> baidu.com` 不应识别为重叠冲突
- `*.baidu.com -> a.b.baidu.com` 不应识别为重叠冲突
- `222.0.0.0/24 -> 222.0.0.1/32` 在协议和端口相交时必须识别为重叠冲突
- 同一 IP 范围但协议不相交或端口不相交时，不应误判为重叠冲突
