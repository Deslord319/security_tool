1. HarmonyOS 权限体系概述

1.1 APL 等级

APL 等级说明典型权限前缀
normal风险最低，安装时自动授予ohos.permission.INTERNET
system_basic中等风险，需 ACL 提升或系统应用ohos.permission.ENTERPRISE_*
system_core最高风险，仅限核心系统服务ohos.permission.MANAGE_*

1.2 授权方式

类型说明适用场景
system_grant安装时自动授权企业管理类权限（本项目全部属此类）
user_grant运行时弹窗向用户请求相机、麦克风、位置等敏感权限

1.3 ACL 机制

当应用 APL 等级低于权限要求时，可通过 ACL（Access Control List）机制申请更高等级的权限。本项目 APL 为 normal，但声明了 app-feature: hos_system_app，通过签名模板中的 acls.allowed-acls 提升至可使用 system_basic 级别权限。

4. 风险评估

风险 1：admin 参数传 null — 高风险

位置：PeripheralService.ets 全部 15 个方法

问题：所有企业级 API 调用均以 null 作为 admin 参数。根据 HarmonyOS MDM Kit 文档，admin 参数用于标识设备管理员应用身份。传入 null 时：

  • 若应用未注册为 MDM 管理员，将触发错误码 9200001（not device admin app）
  • 即使注册为管理员，null 可能导致权限验证路径不完整

影响：所有 15 个 API 方法在未正确注册管理员身份时均会调用失败

风险 2：缺少管理员身份预检 — 中风险

问题：代码未在调用企业 API 前检测当前应用是否已注册为设备管理员。错误仅在 catch 块中通过 hilog 记录，用户界面无法获知失败原因。

影响：用户操作后无反馈或显示不明错误

风险 3：蓝牙白名单空数据歧义 — 低风险

位置：getBluetoothWhitelist（行 156-165）

问题：返回空数组时无法区分"策略未启用"和"白名单为空"两种状态，可能导致 UI 层展示逻辑错误。

5. 代码修正计划

修正 A：增加管理员身份检测

优先级：P0 - 必须修正

说明：在所有企业 API 调用前，调用 adminManager.isSuperAdmin(bundleName) 或 adminManager.isAdminAppActive(admin) 检测当前应用是否为激活的设备管理员。

import { adminManager } from '@kit.MDMKit';

// 在 PeripheralService 中增加静态方法
static async checkAdminActive(): Promise<boolean> {
  try {
    const isAdmin = await adminManager.isSuperAdmin('com.example.securitytool');
    return isAdmin;
  } catch (err) {
    hilog.error(0x0000, TAG, 'Admin check failed: %{public}s', JSON.stringify(err));
    return false;
  }
}

修正 B：页面层增加权限前置验证

优先级：P1 - 建议修正

说明：在外设管理页面 aboutToAppear 生命周期中调用管理员检测，非管理员状态下禁用操作按钮并显示提示信息。

async aboutToAppear() {
  this.isAdmin = await PeripheralService.checkAdminActive();
  if (!this.isAdmin) {
    promptAction.showToast({ message: '应用未注册为设备管理员' });
  }
}

修正 C：统一 admin 参数传递

优先级：P1 - 建议修正

说明：将硬编码的 null admin 参数替换为统一的 Want 对象构造，确保 MDM 框架能正确识别调用方身份。

const ADMIN_WANT: Want = {
  bundleName: 'com.example.securitytool',
  abilityName: 'EnterpriseAdminAbility'
};

// 修正前: restrictions.setDisallowedPolicy(null, ...)
// 修正后: restrictions.setDisallowedPolicy(ADMIN_WANT, ...)

修正 D：蓝牙空数据语义区分

优先级：P2 - 可选优化

说明：对 getBluetoothWhitelist 返回值增加语义封装，区分"未配置策略"与"白名单为空"两种状态。

interface BluetoothPolicyResult {
  enabled: boolean;
  devices: Array<string>;
}

static getBluetoothWhitelist(): BluetoothPolicyResult {
  try {
    const list = bluetoothManager.getAllowedBluetoothDevices(null);
    return { enabled: true, devices: list };
  } catch (err) {
    if (err.code === 9200002) {
      return { enabled: false, devices: [] };
    }
    throw err;
  }
}
