日志管理模块

基于 SecurityAudit API 的安全审计事件订阅、展示、筛选与导出功能架构设计

HarmonyOS SecurityTool | 版本 1.0 | 2026-02-26

1. 模块概述

功能定位

日志管理模块是 SecurityTool 应用的安全审计中心，为企业 IT 管理员提供设备安全事件的实时监控、历史查询和审计追溯能力。

核心业务场景

场景 描述 用户价值
实时监控 订阅安全审计事件，实时展示文件操作、网络连接、USB 访问等日志 及时发现安全威胁
多维筛选 按事件类型、时间范围、进程名/PID、关键词过滤日志 快速定位问题
日志导出 将筛选后的日志导出为文件，支持离线审计 合规审计留痕
存储管理 配置日志保留天数和存储空间上限，自动清理过期日志 控制存储成本

2. 需求分析

3. 技术挑战

⚠ C API 调用挑战
SecurityAudit 是 C API (security_audit.h)，非 ArkTS API。需要通过 NAPI 封装或使用 Native 模块调用。

系统能力: SystemCapability.Security.SecurityAudit
挑战 描述 解决方案
C API 调用 ArkTS 无法直接调用 C 函数 NAPI 封装 / Native 模块
实时事件订阅 需要实现事件监听回调机制，处理异步事件流 EventEmitter 模式 + 状态管理
大数据量展示 日志数据量可能很大，影响渲染性能 分页 / 虚拟滚动 / LazyForEach
本地存储管理 需要管理日志文件生命周期 定时清理任务 + 文件系统 API

日志条目接口

// 日志条目接口
export interface LogEntry {
  id: string                    // 唯一标识
  timestamp: number             // 时间戳 (毫秒)
  eventType: LogEventType       // 事件类型
  detail: string                // 事件详情
  processName: string           // 进程名称
  pid: number                   // 进程 ID
  uid?: number                  // 用户 ID
  filePath?: string             // 文件路径（文件操作事件）
  remoteAddr?: string           // 远程地址（网络事件）
  result: LogEventResult        // 事件结果
}

// 事件结果枚举
export enum LogEventResult {
  SUCCESS = 'success',          // 成功/正常
  BLOCKED = 'blocked',          // 拦截
  FAILED = 'failed'             // 失败
}

筛选条件接口

// 筛选条件接口
export interface LogFilter {
  eventTypes?: LogEventType[]   // 事件类型（多选）
  startTime?: number            // 开始时间戳
  endTime?: number              // 结束时间戳
  keyword?: string              // 关键词（进程名/文件路径/PID）
  processName?: string          // 进程名称
  pid?: number                  // 进程 ID
}

// 分页请求参数
export interface LogPageRequest {
  filter: LogFilter
  page: number                  // 页码（从 1 开始）
  pageSize: number              // 每页条数
}

// 分页响应
export interface LogPageResponse {
  items: LogEntry[]             // 日志列表
  total: number                 // 总条数
  page: number                  // 当前页码
  pageSize: number              // 每页条数
}

6. Service 层 API 设计

LogAuditService

方法 参数 返回值 说明
initialize() - Promise<boolean> 初始化审计客户端
subscribe() eventTypes: LogEventType[] Promise<boolean> 订阅指定事件类型
unsubscribe() eventTypes: LogEventType[] Promise<boolean> 取消订阅
getLogs() request: LogPageRequest Promise<LogPageResponse> 获取日志列表（分页）
exportLogs() filter: LogFilter, format: string Promise<string> 导出日志文件，返回文件路径
onEvent() callback: (entry: LogEntry) => void void 注册事件回调
destroy() - void 销毁客户端，释放资源

LogStorageService

方法 参数 返回值 说明
getConfig() - Promise<LogStorageConfig> 获取存储配置
setRetentionDays() days: number Promise<boolean> 设置保留天数
getUsage() - { used: number, total: number } 获取存储使用情况
clearLogs() before?: Date Promise<boolean> 清除日志（可选：指定日期之前）
cleanupExpired() - Promise<number> 清理过期日志，返回清理数量

8. View 层设计

组件状态管理

@Component
export struct LogManagePage {
  // 日志列表状态
  @State logs: LogEntry[] = []
  @State total: number = 0
  @State currentPage: number = 1
  @State pageSize: number = 20
  @State isLoading: boolean = false
  
  // 筛选条件状态
  @State selectedEventType: string = 'all'
  @State startDate: string = ''
  @State endDate: string = ''
  @State keyword: string = ''
  
  // 存储管理状态
  @State storageConfig: LogStorageConfig = DEFAULT_LOG_STORAGE_CONFIG
  
  // 主题
  @StorageLink(STORAGE_KEY_THEME) currentTheme: ThemeMode = ThemeMode.LIGHT
  
  // 生命周期
  aboutToAppear() {
    this.initService()
    this.loadLogs()
    this.loadStorageConfig()
  }
  
  aboutToDisappear() {
    LogAuditService.destroy()
  }
}
