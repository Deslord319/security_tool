深色模式设计方案

HarmonyOS 安全管理中心 - 主题切换功能设计规格

需求概述

用户可以自定义选择应用的显示模式（深色/浅色），同时支持跟随系统设置自动切换。

核心需求：

  • 当用户未手动选择模式时，应用内跟随系统主题
  • 当用户手动选择模式时，用户选择为最高优先级，忽略系统变化
  • 用户选择需要持久化保存，重启应用后生效

UI 设计

1. 设置入口位置

主题设置位于 工具设置 页面，作为独立的配置区块。

2. 交互组件设计

采用三选一的分段选择器（Segmented Control）设计，直观展示当前选择状态。

3. 选项说明

跟随系统
应用主题随系统设置自动切换。当系统切换深色/浅色模式时，应用实时响应变化。这是默认选项。
浅色模式
强制使用浅色主题，不受系统设置影响。适合在明亮环境下使用或偏好浅色界面的用户。
深色模式
强制使用深色主题，不受系统设置影响。适合在暗光环境下使用，减少眼睛疲劳。

功能逻辑

1. 优先级规则

用户手动选择（最高优先级）
当用户选择"浅色"或"深色"时，应用固定使用该主题，完全忽略系统配置变化。
跟随系统（默认行为）
当用户选择"跟随系统"时，应用监听系统配置变化，实时同步系统的深色/浅色模式。

数据模型

1. 状态定义

// 主题模式
enum ThemeMode {
  LIGHT = 'light',
  DARK = 'dark'
}

// 用户偏好
enum ThemePreference {
  SYSTEM = 'system',  // 跟随系统（默认）
  MANUAL = 'manual'   // 手动设置
}

2. 存储结构

存储位置 Key Value 说明
AppStorage currentTheme 'light' | 'dark' 当前实际显示的主题，UI 组件通过 @StorageLink 绑定
AppStorage themePreference 'system' | 'manual' 用户偏好类型，决定是否响应系统变化
Preferences theme_mode 'light' | 'dark' 持久化存储的主题值
Preferences theme_preference 'system' | 'manual' 持久化存储的偏好类型

技术实现

1. ThemeManager 核心方法

class ThemeManager {
  // 初始化：同步检测系统主题 + 异步加载用户偏好
  init(context: UIAbilityContext): void

  // 设置主题（切换为手动模式）
  async setTheme(mode: ThemeMode): Promise<void>

  // 设置偏好（切换跟随系统/手动）
  async setPreference(pref: ThemePreference): Promise<void>

  // 系统配置变化回调
  onConfigurationUpdate(): void

  // 重新应用主题（修复初始化时序问题）
  reapplyTheme(): void
}

2. 关键 API

功能 API 模块
检测系统主题 context.config.colorMode @kit.AbilityKit
监听系统变化 onConfigurationUpdate() UIAbility 生命周期
持久化存储 preferences.getPreferences() @kit.ArkData
响应式状态 @StorageLink / AppStorage ArkTS 状态管理

3. 组件绑定示例

@Entry
@Component
struct MainPage {
  // 双向绑定 AppStorage 中的主题值
  @StorageLink('currentTheme') currentTheme: ThemeMode = ThemeMode.LIGHT

  build() {
    Column() {
      // 根据主题切换背景色
    }
    .backgroundColor(AppColors.getColors(this.currentTheme).BG_PAGE)
  }
}

边界情况处理

场景 处理方式
首次安装，无偏好记录 默认使用"跟随系统"，检测并应用系统当前主题
Preferences 读取失败 回退到"跟随系统"模式，使用系统主题
系统 colorMode 为 NOT_SET 使用上次检测到的系统主题，避免误判为浅色
应用启动时系统配置变化 initSync 在页面加载前完成，reapplyTheme 作为安全兜底

测试用例

  1. 默认行为：新安装应用，系统为深色模式 → 应用显示深色主题
  2. 跟随系统切换：保持"跟随系统"，系统从浅色切深色 → 应用自动切换
  3. 手动选择优先：选择"浅色模式"，系统切深色 → 应用保持浅色
  4. 切回跟随系统：从"手动深色"切回"跟随系统" → 立即应用系统当前主题
  5. 持久化验证：选择"深色模式"，杀掉应用重启 → 仍为深色模式
  6. 冷启动时序：系统深色，冷启动应用 → 启动瞬间即为深色（无闪烁）
