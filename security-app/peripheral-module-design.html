1. 需求分析（来自 UI 原型）

根据 security-app/index.html 中外设管理页面的 UI 原型，该模块包含 4 个功能 Tab：

Tab 功能 核心操作 主要 API 来源
设备连接记录 展示 USB/蓝牙设备的历史连接记录 查看、导出 本地数据存储
接口管控 全局禁用/启用 USB、蓝牙、Wi-Fi、HDC 接口 开关切换 @ohos.enterprise.restrictions
设备管控 针对已知设备设置单独的允许/禁用策略 策略变更 @kit.MDMKit usbManager
黑白名单 USB 设备白名单(VID/PID)、USB 类型黑名单、蓝牙白名单(MAC) 增删查 @kit.MDMKit usbManager + bluetoothManager

防火墙模块

  • FirewallService.ets — netFirewall API
  • FirewallPage.ets — 概览 + 开关 + 分类卡片
  • FirewallRulesPage.ets — Tabs + 表格 CRUD
  • AddRuleDialog.ets — 表单弹窗
  • RuleTypeItem.ets — 列表项组件

外设管理模块

  • PeripheralService.ets — restrictions + MDMKit API
  • PeripheralPage.ets — 4 Tab 页整合 (设备记录/接口/设备/黑白名单)
  • Tab 内直接使用 @Builder 构建各子页
  • AddWhitelistDialog.ets / AddBlacklistDialog.ets
  • InterfaceControlItem.ets — 接口管控列表项
i
设计决策：防火墙模块拆分为两个 View（FirewallPage + FirewallRulesPage），外设管理模块因为 4 个 Tab 属于同一页面上下文，建议合并为单个 PeripheralPage.ets，内部用 Tabs + @Builder 分拆，更符合 UI 原型的交互模式。

4.3 辅助函数

export function getPeripheralTypeLabel(type: PeripheralType): string {
  switch (type) {
    case PeripheralType.USB_STORAGE: return 'USB 存储设备'
    case PeripheralType.USB_CAMERA:  return 'USB 摄像头'
    case PeripheralType.USB_PRINTER: return 'USB 打印机'
    case PeripheralType.USB_OTHER:   return 'USB 其他'
    case PeripheralType.BLUETOOTH:   return '蓝牙设备'
    default: return '未知'
  }
}

export function getUsbStoragePolicyLabel(policy: UsbStoragePolicy): string {
  switch (policy) {
    case UsbStoragePolicy.READ_WRITE: return '读写'
    case UsbStoragePolicy.READ_ONLY:  return '只读'
    case UsbStoragePolicy.DISABLED:   return '禁止访问'
    default: return '未知'
  }
}

5. Service 层设计

参照 FirewallService.ets 的静态类 + async + try-catch + hilog 模式，PeripheralService.ets 需要封装 三组 API：

5.1 接口管控 (restrictions)

import { restrictions } from '@ohos.enterprise.restrictions'
import { hilog } from '@kit.PerformanceAnalysisKit'

const TAG = 'PeripheralService'
const DOMAIN = 0x0002

export class PeripheralService {

  // ========== 接口管控 (restrictions) ==========

  // 设置接口禁用策略
  static async setInterfaceDisabled(feature: string, disallow: boolean): Promise<boolean> {
    try {
      restrictions.setDisallowedPolicy(null, feature, disallow)
      hilog.info(DOMAIN, TAG, `setInterfaceDisabled: ${feature} = ${disallow}`)
      return true
    } catch (err) {
      hilog.error(DOMAIN, TAG, `setInterfaceDisabled failed: ${JSON.stringify(err)}`)
      return false
    }
  }

  // 获取接口禁用状态
  static getInterfaceDisabled(feature: string): boolean {
    try {
      return restrictions.getDisallowedPolicy(null, feature)
    } catch (err) {
      hilog.error(DOMAIN, TAG, `getInterfaceDisabled failed: ${JSON.stringify(err)}`)
      return false
    }
  }
  ...

7. View 层设计（4 个 Tab 页）

PeripheralPage.ets 内部通过 Tabs 组件切换 4 个子页面，每个 Tab 用 @Builder 构建。

7.1 整体结构

@Component
export struct PeripheralPage {
  @State currentTab: number = 0
  @State isLoading: boolean = false

  // 接口管控状态
  @State usbDisabled: boolean = false
  @State btDisabled: boolean = false
  @State wifiDisabled: boolean = false
  @State hdcDisabled: boolean = false
  @State usbStoragePolicy: number = 0

  // 黑白名单数据
  @State usbWhitelist: UsbDeviceId[] = []
  @State usbBlacklist: UsbDeviceType[] = []
  @State btWhitelist: string[] = []

  aboutToAppear() { this.loadAllData() }

  build() {
    Column() {
      // 页面头部: 标题 + 搜索框
      Row() { ... }

      // Tab 切换
      Tabs({ barPosition: BarPosition.Start }) {
        TabContent() { this.deviceRecordsBuilder() }
          .tabBar(this.tabBuilder('设备连接记录', 0))
        TabContent() { this.interfaceControlBuilder() }
          .tabBar(this.tabBuilder('接口管控', 1))
        TabContent() { this.deviceControlBuilder() }
          .tabBar(this.tabBuilder('设备管控', 2))
        TabContent() { this.blackWhiteListBuilder() }
          .tabBar(this.tabBuilder('黑白名单', 3))
      }
    }
  }
}

7.2 Tab 功能说明

Tab 1: 设备连接记录 (@Builder deviceRecordsBuilder)

对应 UI 原型中的设备连接表格。使用 List + ForEach 展示设备记录。

列数据源
设备名称deviceName
设备类型getPeripheralTypeLabel(deviceType)
设备标识deviceId (VID:PID 或 MAC)
连接时间connectTime
策略allow/deny Tag 标签

Tab 2: 接口管控 (@Builder interfaceControlBuilder)

对应 UI 原型中的接口开关列表。使用 InterfaceControlItem 组件渲染每一行。

接口API feature控件类型对应 API
USB 接口"usb"Select (按策略生效/全局禁用)restrictions.setDisallowedPolicy
USB 存储设备-Select (只读/读写/禁止)usbManager.setUsbStorageDeviceAccessPolicy
蓝牙"bluetooth"Select (启用/禁用)restrictions.setDisallowedPolicy
Wi-Fi"wifi"Select (启用/禁用)restrictions.setDisallowedPolicy
HDC"hdc"Select (禁用/启用)restrictions.setDisallowedPolicy

Tab 3: 设备管控 (@Builder deviceControlBuilder)

对应 UI 原型中的已知设备策略管理表格。展示已连接过的 USB 设备，允许为每个设备单独配置策略。

Tab 4: 黑白名单 (@Builder blackWhiteListBuilder)

对应 UI 原型中的三段式黑白名单。包含三个子区域：

子区域标识字段增删 API
USB 白名单 (按设备 ID) vendorId + productId usbManager.addAllowedUsbDevices / removeAllowedUsbDevices
USB 黑名单 (按设备类型) baseClass + subClass + protocol usbManager.addDisallowedUsbDevices / removeDisallowedUsbDevices
蓝牙白名单 (按 MAC) MAC 地址 bluetoothManager.addAllowedBluetoothDevices / removeAllowedBluetoothDevices

8. 组件设计（弹窗 + 列表项）

8.1 InterfaceControlItem.ets

参照 RuleTypeItem.ets 的 @Prop 输入 + 回调模式：

@Component
export struct InterfaceControlItem {
  @Prop label: string = ''
  @Prop description: string = ''
  @Prop options: string[] = []      // ['启用', '禁用'] 或 ['读写', '只读', '禁止访问']
  @Prop selectedIndex: number = 0
  onSelect: (index: number) => void = () => {}

  build() {
    Row() {
      Column() {
        Text(this.label)        // 接口名称
        Text(this.description)  // 接口描述
      }
      Select(this.options)       // 下拉选择
    }
  }
}

8.2 AddWhitelistDialog.ets

参照 AddRuleDialog.ets 的 @CustomDialog 模式。支持两种模式：USB（VID + PID + 备注）和蓝牙（MAC + 备注）。

@CustomDialog
export struct AddWhitelistDialog {
  controller: CustomDialogController
  @Prop mode: 'usb' | 'bluetooth' = 'usb'

  // USB 模式字段
  @State vendorId: string = ''
  @State productId: string = ''

  // 蓝牙模式字段
  @State macAddress: string = ''

  // 公共
  @State remark: string = ''
  onConfirm: (...) => void = () => {}

  build() {
    Column() {
      Text(mode === 'usb' ? '添加 USB 设备白名单' : '添加蓝牙设备白名单')
      if (this.mode === 'usb') {
        TextInput({ placeholder: 'Vendor ID (例: 0x2B7E)' })
        TextInput({ placeholder: 'Product ID (例: 0xB5A2)' })
      } else {
        TextInput({ placeholder: 'MAC 地址 (例: AA:BB:CC:DD:EE:01)' })
      }
      TextInput({ placeholder: '备注' })
      // 取消 + 添加 按钮
    }
  }
}

i
与防火墙模块的区别：防火墙模块的 firewallEnabled 需要在 Dashboard 中展示，因此通过 @State/@Prop 链从 MainPage 透传。外设管理模块的状态不需要跨页面共享，因此 PeripheralPage 可以完全自治，内部通过 Service 层直接读写状态。
