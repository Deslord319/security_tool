# 需求 说明 优先级
1 去除系统标题栏 隐藏系统默认的窗口装饰栏（包含应用图标、应用名称），使应用内容延伸到标题栏区域，获得更大的可用空间 P0

3. 核心 API 参考

window.Window.setWindowDecorVisible(isVisible: boolean): Promise<void>
设置窗口标题栏（装饰栏）是否可见。仅对存在标题栏和三键区的窗口形态生效（2in1 / 平板自由窗口模式）。
isVisible: boolean
true - 显示标题栏（默认）；false - 隐藏标题栏（应用图标+名称不可见，但三键区仍然保留）
关键行为：调用 setWindowDecorVisible(false) 后，系统标题栏的应用图标和名称被隐藏， 应用页面内容向上拓展至原标题栏区域。但 三键区（最小化/最大化/关闭按钮）仍然保留， 悬浮在应用内容上方。同时 窗口拖拽功能保持不变，用户仍可通过顶部区域拖动窗口。
API 11+ 2in1 / Tablet
window.Window.setWindowDecorHeight(height: number): void
设置窗口标题栏（装饰栏）的高度，仅支持 2in1 设备或平板设备的自由多窗模式。通常与 setWindowDecorVisible(false) 配合使用，控制三键区的高度。
height: number
标题栏高度（单位 vp）。设置后三键区将使用此高度。推荐值：0（完全隐藏装饰栏区域）或自定义值如 48 / 56（匹配应用内标题栏高度）
API 11+ 2in1 / Tablet
window.Window.setTitleAndDockHoverShown(isTitleHoverShown: boolean, isDockHoverShown: boolean): Promise<void>
设置主窗口进入全屏沉浸式时，鼠标 Hover 到热区上方是否显示标题栏和 Dock 栏。
isTitleHoverShown: boolean
true - Hover 热区时显示标题栏；false - 不显示
isDockHoverShown: boolean
true - Hover 热区时显示 Dock；false - 不显示
API 14+ 仅全屏沉浸式
window.Window.setWindowLayoutFullScreen(isLayoutFullScreen: boolean): Promise<void>
设置窗口是否以全屏布局模式显示，即应用页面是否延伸到状态栏和导航栏区域。
isLayoutFullScreen: boolean
true - 全屏布局；false - 非全屏布局（默认）
注意：本需求场景为 2in1 自由窗口模式，不需要使用 setWindowLayoutFullScreen。 该 API 主要用于手机/平板的沉浸式状态栏场景。2in1 设备应使用 setWindowDecorVisible + setWindowDecorHeight 组合。
API 9+ 所有设备

4. 方案设计

推荐方案：setWindowDecorVisible(false) + setWindowDecorHeight 推荐

通过隐藏系统装饰栏的应用图标和名称，保留三键区（最小化/最大化/关闭），并设置装饰栏高度与应用内标题栏对齐， 实现无缝融合效果。系统自动保留顶部拖拽热区，无需额外处理拖拽逻辑。

页面内容区

实施步骤

  1. 修改 EntryAbility.ets - 隐藏系统装饰栏
    在 onWindowStageCreate 中获取窗口实例，调用 setWindowDecorVisible(false) 隐藏标题栏， 调用 setWindowDecorHeight(56) 设置三键区高度与应用内标题栏对齐。
  2. 调整 MainPage.ets - 适配标题栏布局
    内容区标题栏右侧需为三键区预留空间（约 120px），避免三点菜单按钮与系统三键区重叠。
  3. 调整 SideBar.ets - Logo 区域适配
    SideBar 顶部 Logo 区域可能需要微调上边距，确保在没有系统标题栏时视觉间距合理。
  4. 编译、签名、安装、验证
    在 2in1 设备上验证：标题栏已隐藏、三键区正常工作、窗口可拖拽、自由窗口可缩放。

5. 核心代码变更

EntryAbility.ets 核心修改

onWindowStageCreate 修改

import { window } from '@kit.ArkUI';
import { hilog } from '@kit.PerformanceAnalysisKit';

// 在 onWindowStageCreate 中添加窗口装饰栏设置
onWindowStageCreate(windowStage: window.WindowStage): void {
  // 获取主窗口实例
  let mainWindow: window.Window = windowStage.getMainWindowSync();

  // 1. 隐藏系统标题栏（应用图标+名称），保留三键区
  try {
    mainWindow.setWindowDecorVisible(false);
    hilog.info(DOMAIN, 'testTag', 'Window decor hidden successfully');
  } catch (err) {
    hilog.error(DOMAIN, 'testTag',
      'Failed to hide window decor: %{public}s', JSON.stringify(err));
  }

  // 2. 设置装饰栏高度，与应用内标题栏高度对齐
  try {
    mainWindow.setWindowDecorHeight(56);
    hilog.info(DOMAIN, 'testTag', 'Window decor height set to 56');
  } catch (err) {
    hilog.error(DOMAIN, 'testTag',
      'Failed to set decor height: %{public}s', JSON.stringify(err));
  }

  // 3. 加载页面（原有逻辑不变）
  windowStage.loadContent('pages/MainPage', (err) => {
    if (err.code) {
      hilog.error(DOMAIN, 'testTag',
        'Failed to load content: %{public}s', JSON.stringify(err));
      return;
    }
    ThemeManager.getInstance().reapplyTheme();
  });
}
关键点：
  • getMainWindowSync() 同步获取主窗口，无需 await
  • setWindowDecorVisible(false) 隐藏图标和名称，三键区自动保留
  • setWindowDecorHeight(56) 让三键区高度与应用内标题栏（56vp）对齐
  • 系统自动处理窗口拖拽热区，无需手动实现拖拽逻辑
MainPage.ets 布局调整

标题栏右侧预留三键区空间

隐藏系统标题栏后，三键区悬浮在应用内容右上角。应用内标题栏的右侧需为三键区预留空间， 防止三点菜单按钮被遮挡。 

// 顶部标题栏 - 修改右侧 padding 为三键区预留空间
Row() {
  Text(this.getPageTitle())
    .fontSize(AppStyles.FONT_SIZE_TITLE)
    .fontWeight(FontWeight.Medium)
    .fontColor(AppColors.getColors(this.currentTheme).TEXT_PRIMARY)

  Blank()

  // 三点菜单按钮
  Button() {
    Text('⋮')
      .fontSize(20)
      .fontWeight(FontWeight.Bold)
      .fontColor(AppColors.getColors(this.currentTheme).TEXT_PRIMARY)
  }
  .onClick(() => { this.showMenu = !this.showMenu })
}
.width('100%')
.height(56)
.padding({
  left: AppStyles.PADDING_LG,
  right: 140  // 原为 PADDING_LG(24)，增加到 140 为三键区预留空间
})
三键区宽度参考：2in1 设备三键区宽度约为 100-120px（含最小化、最大化、关闭三个按钮）。 建议右侧 padding 设置为 ~140vp（三键区宽度 + 三点菜单按钮宽度 + 间距），实际数值可在设备上微调。
MainPage.ets 菜单位置

下拉菜单定位调整

// 下拉菜单 - 右侧 margin 需加大，避免与三键区重叠
if (this.showMenu) {

6. 风险评估

低风险
窗口拖拽

setWindowDecorVisible(false) 隐藏标题栏后，系统自动保留顶部拖拽热区。 无需手动实现拖拽逻辑。这是系统级行为，不会丢失。

低风险
三键区功能

三键区（最小化/最大化/关闭）在隐藏标题栏后自动悬浮在内容上方，功能不受影响。

中风险
三键区遮挡

三键区悬浮在内容上方，可能遮挡右上角的 UI 元素。需确保标题栏右侧为三键区预留足够空间。

中风险
API 版本兼容

setWindowDecorVisible 和 setWindowDecorHeight 需要 API 11+。 需用 try-catch 包裹，确保低版本设备优雅降级。
